# SQLRustGo 开发规范

> **重要原则**：绝不删除文档原始内容

---

## 文档操作原则

### 🚫 禁止操作

1. **禁止用 Write 完全覆盖文档**
   - 原因：会删除原始信息和历史记录
   - 替代方案：使用 SearchReplace 做局部修改

2. **禁止未经确认删除内容**
   - 原因：文档原始内容需要保留
   - 替代方案：先备份或与用户确认

### ✅ 正确操作

1. **使用 SearchReplace 修改**
   - 只修改需要变更的部分
   - 保持文档其余部分不变

2. **修改前先 Read 完整内容**
   - 了解文档当前状态
   - 确认修改位置

3. **保留变更历史**
   - 文档底部添加变更记录
   - 记录修改日期和原因

---

## 执行流程

```
1. 读取文档 → 2. 分析内容 → 3. 局部修改 → 4. 提交变更
```

---

## 违反处理

如果误操作：
1. 立即使用 `git reset --hard` 恢复
2. 向用户道歉并说明原因
3. 重新按照正确流程操作
