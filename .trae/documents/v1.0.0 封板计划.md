## 核心纪律规则

### 四条铁律
1. **版本节奏固定** - 每 2 周一个 patch，每季度一个 minor
2. **CI 必须阻断** - fmt/clippy/unwrap/覆盖率检查
3. **PR 必须小** - ≤ 400 行改动，≤ 1 个模块
4. **发布优先于优化** - 可发布 > 更优雅

### API 冻结原则
- 禁止删除 public 方法
- 禁止修改函数参数
- 禁止修改返回类型
- 允许新增 API

### 技术债分类
- A 类：影响稳定性（必须立即修复）
- B 类：影响可维护性（记录在 backlog）
- C 类：优化类（延后到 2.0）
