## What

SQL Semantic Governance Guide for SQLRustGo - full-stack SQL semantic correctness framework.

## Why

SQLRustGo 2.8.0 has confirmed semantic defects in Executor and Parser layers that violate SQL standard behavior. This document establishes a systematic governance framework to identify, prioritize, and fix these issues.

## Confirmed Defects

### Executor (High Severity)
- E-1: NULL = NULL returns true (should be UNKNOWN/false in WHERE)
- E-2: execute_select_with_join does not apply WHERE clause
- E-3: COUNT(col) does not skip NULL

### Executor (Medium Severity)
- E-4: SUM all-NULL returns 0 instead of NULL
- E-5: JOIN NULL key matching via format!("{:?}") 
- E-6: Aggregate only handles Integer, not Float
- E-7: NOT/AND/OR incomplete three-valued logic

### Parser
- P-1: IS not defined as independent Token
- P-2: IS NULL has no structured AST node
- P-3: col = NULL not rejected

## Framework

- Layered governance model (Parser -> Planner -> Executor -> Storage)
- Semantic matrix methodology (Feature x Boundary x Combination)
- Critical path review (JOIN > WHERE > GROUP BY > Aggregate > Subquery)
- Semantic guard test requirements
- Single semantic entry point convergence plan
- Phased evolution roadmap (Phase 1/2/3 for NULL/3VL)
- AI-assisted governance prompt templates
