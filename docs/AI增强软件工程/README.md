# AI 增强软件工程

> **核心理念**：人类建立流程，AI 协作执行。工具不是重点，思路才是关键。

---

## 一、为什么需要 AI 增强软件工程？

### 1.1 传统软件开发的瓶颈

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        传统开发的困境                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   人类开发者面临的挑战：                                                      │
│                                                                              │
│   📋 规划瓶颈                                                                │
│      • 难以全面考虑所有因素                                                  │
│      • 架构设计容易遗漏细节                                                  │
│      • 文档编写耗时且容易过时                                                │
│                                                                              │
│   💻 执行瓶颈                                                                │
│      • 重复性工作消耗精力                                                    │
│      • 代码审查难以全面                                                      │
│      • 测试覆盖难以完善                                                      │
│                                                                              │
│   📊 总结瓶颈                                                                │
│      • 项目复盘流于形式                                                      │
│      • 经验教训难以沉淀                                                      │
│      • 知识传承效率低下                                                      │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 AI 增强的核心价值

| 阶段 | AI 的价值 | 人类的价值 |
|:-----|:----------|:-----------|
| **规划** | 快速分析、全面考虑、生成方案 | 决策方向、设定约束、评估风险 |
| **执行** | 高效编码、自动测试、持续集成 | 审核把关、处理异常、调整策略 |
| **总结** | 整理文档、提取经验、生成报告 | 提炼洞察、制定改进、知识传承 |

## 二、AI 工具链的正确使用思路

### 2.1 工具分层模型

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        AI 工具分层模型                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   第 1 层：规划与分析层（大脑层）                                             │
│   ─────────────────────────────────                                         │
│   • ChatGPT (网页版)     → 架构设计、流程规划、概念解释                       │
│   • Claude (网页版)      → 深度推理、文档写作、多轮对话                       │
│   • GitHub Copilot      → 代码分析、质量评估、技术细节                       │
│                                                                              │
│   第 2 层：执行与协作层（手脚层）                                             │
│   ─────────────────────────────────                                         │
│   • Claude Code         → IDE 内编码、重构、调试                             │
│   • OpenCode            → 命令行自动化、批量处理                             │
│   • Codex CLI           → 代码生成、API 调用                                 │
│                                                                              │
│   第 3 层：协调与管理层（调度层）                                             │
│   ─────────────────────────────────                                         │
│   • OpenClaw            → 多 Agent 协调、飞书集成、定时任务                   │
│   • TRAE IDE            → 文档编写、项目分析、对话记录                        │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 选择工具的原则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        工具选择原则                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   问自己三个问题：                                                            │
│                                                                              │
│   1. 这是什么类型的任务？                                                    │
│      • 需要深度思考？ → 用 Claude / ChatGPT                                  │
│      • 需要快速执行？ → 用 Claude Code / OpenCode                            │
│      • 需要多人协作？ → 用 OpenClaw + 飞书                                   │
│                                                                              │
│   2. 我需要什么样的输出？                                                    │
│      • 方案和规划？ → 用规划层工具                                           │
│      • 代码和测试？ → 用执行层工具                                           │
│      • 文档和报告？ → 用协调层工具                                           │
│                                                                              │
│   3. 我有多少时间？                                                          │
│      • 时间充裕？ → 深度交互，多轮对话                                       │
│      • 时间紧迫？ → 自动化执行，快速迭代                                     │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          关键经验总结                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   1. 规划先行，工具在后                                                      │
│      • 先想清楚要做什么，再选择合适的工具                                    │
│      • 不要被工具限制思路                                                    │
│                                                                              │
│   2. 流程比工具更重要                                                        │
│      • 建立清晰的协作流程                                                    │
│      • 定义明确的角色和职责                                                  │
│      • 设定可量化的质量门禁                                                  │
│                                                                              │
│   3. 人类决策，AI 执行                                                       │
│      • 人类负责方向、审核、调整                                              │
│      • AI 负责执行、汇报、交叉审核                                           │
│      • 关键决策必须人工确认                                                  │
│                                                                              │
│   4. 持续迭代，不断优化                                                      │
│      • 每个版本都要总结经验                                                  │
│      • 根据反馈调整流程                                                      │
│      • 文档要随项目演进更新                                                  │
│                                                                              │
│   5. 对话记录是宝贵资产                                                      │
│      • 记录关键决策和理由                                                    │
│      • 整理经验教训                                                          │
│      • 便于后续复盘和传承                                                    │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 六、快速开始

### 6.1 如果你想规划一个新项目

```
1. 打开 ChatGPT 或 Claude 网页版
2. 粘贴你的项目想法或现有代码链接
3. 要求 AI 帮你分析、规划、设计
4. 记录对话，形成规划文档
```

### 6.2 如果你想建立 AI 协作流程

```
1. 阅读 AI协作开发教程.md
2. 配置 OpenClaw 和飞书群聊
3. 设置 GitHub 多角色权限
4. 开始 Issue 驱动的迭代开发
```

### 6.3 如果你想总结和优化项目

```
1. 使用 GitHub Copilot 分析项目
2. 使用 TRAE IDE 补齐文档
3. 整理对话记录和经验教训
4. 规划下一版本改进方向
```
