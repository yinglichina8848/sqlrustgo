# 文档中文化推进说明

## 目标

为 SQLRustGo 项目建立“面向学生可读”的中文文档体系：

- 保留原始技术信息，不删除历史内容。
- 将说明性文字统一为中文。
- 代码块、命令、分支名、路径保持原样，避免破坏可执行性。

## 当前进展（2026-03-07）

已优先完成核心文档中文化与口径统一：

- 根目录入口与版本状态文档
- v1.2.0 版本计划与阶段治理文档
- 长期路线图与测试计划关键段落
- 统一当前状态为 `alpha/v1.2.0`，当前开发分支为 `develop/v1.2.0`

## 执行规则

1. 历史阶段（如 `v1.2.0-draft`）保留为“历史记录”。
2. 当前状态只使用最新口径，避免学生混淆。
3. 术语处理遵循“中文优先，保留英文缩写”：
   - 发布候选（RC）
   - 一般可用（GA）
   - 成本优化器（CBO）
4. 不翻译代码示例、命令、配置键名。

## 后续分批计划

1. 第一批：`docs/releases/`（版本文档）
2. 第二批：`docs/governance/`（治理文档）
3. 第三批：`docs/architecture/`（架构文档）
4. 第四批：教学目录与历史归档目录

## 验收标准

- 学生阅读时无需依赖英文即可理解流程与规则。
- 版本状态、分支命名、阶段门禁在所有入口文档中一致。
- 文档内部链接可正常跳转，且不破坏命令可执行性。
