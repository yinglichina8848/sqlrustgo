# EXPLAIN ANALYZE Enhancement Design

> **Version**: v1.9.0
> **Date**: 2026-03-26
> **Status**: Design

---

## 1. Overview

Enhance EXPLAIN ANALYZE to provide detailed execution-time profiling, including operator-level timing, row counts, memory usage, and I/O statistics.

## 2. Current State

- Basic EXPLAIN support exists in parser
- Simple plan output without timing

## 3. Target Output Format

```sql
EXPLAIN ANALYZE SELECT * FROM users WHERE id = 1;

┌─────────────────────────────────────────────────────────────────┐
│                      Execution Plan                             │
├─────────────────────────────────────────────────────────────────┤
│  ├─ Seq Scan on users (cost=0.00..5.00 rows=1 width=16)        │
│  │    Output: id, name                                         │
│  │    Filter: (id = 1)                                         │
│  │    Actual Time: 0.015..0.020 rows=1 loops=1                │
│  │    Buffers: shared hit=1                                    │
│  │                                                             │
│  Planning Time: 0.123 ms                                        │
│  Execution Time: 0.456 ms                                      │
└─────────────────────────────────────────────────────────────────┘
```

## 4. Components

### 4.1 Plan Profiler

- Collect per-operator timing
- Track row counts through pipeline
- Record loop counts for repeated scans

### 4.2 Runtime Metrics

- Wall clock time per operator
- CPU time estimation
- Memory allocation tracking
- Buffer cache hits/misses

### 4.3 Output Formatter

- Tree-style plan visualization
- Cost estimation display
- Actual vs estimated comparison

## 5. Implementation

### 5.1 Changes Required

1. **Parser Enhancement** (`crates/parser`)
   - Add ANALYZE flag to ExplainStatement

2. **Executor Enhancement** (`crates/executor`)
   - Add profiling infrastructure
   - Instrument all operators
   - Collect runtime statistics

3. **Output Formatting** (`src/lib.rs`)
   - Human-readable plan display
   - JSON output option

### 5.2 Key Data Structures

```rust
#[derive(Debug, Clone)]
pub struct OperatorProfile {
    pub operator_type: String,
    pub relation_name: Option<String>,
    pub alias: Option<String>,
    
    // Cost estimates
    pub startup_cost: f64,
    pub total_cost: f64,
    pub plan_rows: u64,
    pub plan_width: u64,
    
    // Actual runtime
    pub actual_time: f64,
    pub actual_rows: u64,
    pub loops: u64,
    
    // Buffers
    pub shared_hit: u64,
    pub shared_read: u64,
    pub shared_dirtied: u64,
}

#[derive(Debug, Clone)]
pub struct ExplainResult {
    pub plan: Vec<OperatorProfile>,
    pub planning_time_ms: f64,
    pub execution_time_ms: f64,
}
```

## 6. Operators to Instrument

| Operator | Metrics |
|----------|---------|
| Seq Scan | rows, loops, buffers |
| Index Scan | index name, rows, loops |
| Hash Join | build size, probe rows |
| Merge Join | merge keys, input rows |
| Sort | sort method, memory used |
| Aggregate | group by keys, hash size |
| Limit/Offset | rows returned |

## 7. Success Criteria

- [ ] Execute EXPLAIN ANALYZE without crashing
- [ ] Show per-operator timing
- [ ] Display row counts and loops
- [ ] Show buffer statistics
- [ ] Compare estimated vs actual
- [ ] Format output as tree
