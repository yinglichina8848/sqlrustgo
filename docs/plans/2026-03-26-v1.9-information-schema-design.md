# INFORMATION_SCHEMA Support Design

> **Version**: v1.9.0
> **Date**: 2026-03-26
> **Status**: Design

---

## 1. Overview

Implement INFORMATION_SCHEMA standard to provide metadata access to database objects (tables, columns, indexes, views) in compliance with SQL standard.

## 2. Target Schema

### 2.1 Core Tables

| Table | Description |
|-------|-------------|
| SCHEMATA | Database schemas |
| TABLES | Table metadata |
| COLUMNS | Column metadata |
| VIEWS | View definitions |
| INDEXES | Index information |
| TABLE_CONSTRAINTS | Constraint definitions |
| REFERENTIAL_CONSTRAINTS | Foreign key relationships |

### 2.2 Schema Design

```sql
-- SCHEMATA
CREATE TABLE information_schema.schemata (
    schema_name TEXT PRIMARY KEY,
    schema_owner TEXT,
    default_character_set_catalog TEXT,
    default_character_set_schema TEXT,
    default_character_set_name TEXT
);

-- TABLES
CREATE TABLE information_schema.tables (
    table_schema TEXT,
    table_name TEXT,
    table_type TEXT,
    is_insertable_into TEXT,
    commit_action TEXT
);

-- COLUMNS
CREATE TABLE information_schema.columns (
    table_schema TEXT,
    table_name TEXT,
    column_name TEXT,
    ordinal_position INTEGER,
    column_default TEXT,
    is_nullable TEXT,
    data_type TEXT,
    character_maximum_length INTEGER,
    numeric_precision INTEGER,
    numeric_scale INTEGER
);
```

### 3.3 Key APIs

```rust
pub trait InformationSchemaProvider {
    fn get_schemata(&self) -> Vec<SchemaRow>;
    fn get_tables(&self, schema: &str) -> Vec<TableRow>;
    fn get_columns(&self, schema: &str, table: &str) -> Vec<ColumnRow>;
    fn get_indexes(&self, schema: &str, table: &str) -> Vec<IndexRow>;
}

pub struct InformationSchema {
    catalog: CatalogRef,
}

impl InformationSchema {
    pub fn new(catalog: CatalogRef) -> Self;
    pub fn query(&self, table: &str) -> Result<ExecutorResult, Error>;
}
```

## 4. Integration

### 4.1 Storage Layer

- Register as special "information_schema" database
- Hook into catalog for real-time updates

### 4.2 Query Layer

- Parse INFORMATION_SCHEMA.* queries
- Route to virtual tables
- Return standard-compliant results
