## 1. Overview

Implement query statistics system (similar to PostgreSQL's pg_stat_statements) to track query execution metrics, performance patterns, and provide insights for optimization.

## 3. Metrics Collected

### 3.1 Per-Query Statistics

| Metric | Description |
|--------|-------------|
| calls | Number of times executed |
| total_time | Total execution time (ms) |
| min_time | Minimum execution time |
| max_time | Maximum execution time |
| mean_time | Average execution time |
| stddev_time | Standard deviation |
| rows | Total rows returned |
| shared_blks_hit | Cache hits |
| shared_blks_read | Cache misses |
| local_blks_hit | Local cache hits |
| temp_blks_read | Temporary blocks read |

### 3.2 Aggregated Statistics

- Query frequency ranking
- Slow query identification
- I/O intensive queries
- Memory usage patterns

### 4.2 Query Normalization

```rust
pub struct QueryNormalizer {
    replace_numbers: bool,
    replace_strings: bool,
    replace_identifiers: bool,
}

impl QueryNormalizer {
    // Convert:
    //   SELECT * FROM users WHERE id = 123
    // To:
    //   SELECT * FROM users WHERE id = $1
    pub fn normalize(&self, query: &str) -> String;
}
```

### 4.3 Statistics Collection

```rust
#[derive(Debug, Clone)]
pub struct QueryStats {
    pub query: String,           // Normalized query
    pub calls: u64,              // Execution count
    pub total_time: f64,         // Total time (ms)
    pub min_time: f64,            // Min time
    pub max_time: f64,           // Max time
    pub mean_time: f64,          // Mean time
    pub stddev_time: f64,        // Std deviation
    pub rows: u64,               // Total rows
    pub shared_blks_hit: u64,    // Cache hits
    pub shared_blks_read: u64,   // Cache misses
}

pub struct StatsCollector {
    entries: HashMap<u64, QueryStats>,
    max_entries: usize,
    config: StatsConfig,
}

impl StatsCollector {
    pub fn record(&mut self, query: &str, stats: QueryExecutionStats);
    pub fn get_stats(&self) -> Vec<QueryStats>;
    pub fn reset(&mut self);
}
```

### 4.4 System Views

```sql
-- View similar to pg_stat_statements
SELECT * FROM pg_stat_statements 
ORDER BY total_time DESC 
LIMIT 10;

-- Query statistics by table
SELECT * FROM pg_stat_tables;

-- Index usage statistics
SELECT * FROM pg_stat_indexes;
```

## 6. Integration Points

1. **Executor**: Hook into query execution to collect metrics
2. **Storage**: Persist statistics to WAL
3. **INTEGRATION_SCHEMA**: Expose via INFORMATION_SCHEMA

## 7. Success Criteria

- [ ] Normalize queries consistently
- [ ] Collect timing statistics

- [ ] Support query reset
