## 一、Beta 门禁现状（代码核实）

### 1.1 Build 状态

| 状态 | 问题 | 根因 |
|------|------|------|
| ❌ FAIL | `sqlrustgo-transaction` 编译错误 | borrow after move bug |
| ❌ FAIL | `sqlrustgo-sql-corpus` 编译错误 | non-exhaustive match |

### 1.2 详细编译错误

#### 错误 1: transaction/idempotency/wal.rs

```rust
// Line 179-196
let registry = Arc::new(IdempotencyRegistry::new());
let adapter = IdempotencyWalAdapter::new(registry);  // registry moved here

// ... code ...

let state = registry.get_state("txn-1").unwrap().unwrap();  // ERROR: use after move
```

**问题**: `registry` 被移动到 `IdempotencyWalAdapter::new()` 后又被使用。

#### 错误 2: sql-corpus/src/lib.rs

```rust
// Line 884-898
fn execute_transaction(&mut self, stmt: &TransactionStatement) -> Result<(), String> {
    match stmt {
        TransactionStatement::Begin { .. } => Ok(()),
        TransactionStatement::Commit { .. } => Ok(()),
        TransactionStatement::Rollback { .. } => Ok(()),
        TransactionStatement::SetTransaction { .. } => Ok(()),
        TransactionStatement::StartTransaction { .. } => Ok(()),
        TransactionStatement::Savepoint { .. } => Ok(()),
        TransactionStatement::RollbackToSavepoint { .. } => Ok(()),
        TransactionStatement::ReleaseSavepoint { .. } => Ok(()),
        // MISSING: TransactionStatement::BeginIdempotent { .. }
    }
}
```

**问题**: `TransactionStatement::BeginIdempotent` 变体未覆盖。

## 八、总结

### 8.1 核心发现

1. **v3.1.0 Beta 有 2 个编译错误阻塞门禁**
   - `sqlrustgo-transaction`: borrow after move
   - `sqlrustgo-sql-corpus`: non-exhaustive match

2. **功能实现比文档声称更好**
   - 窗口函数：全部实现 ✅
   - MERGE：已实现 ✅

3. **测试覆盖率下降**
   - 整体从 84.18% 降到 78.48%
   - event.rs 和 trigger_eval 完全未测试

4. **MySQL 兼容性估计提升到 ~65/100**
   - 主要得益于窗口函数完整实现
