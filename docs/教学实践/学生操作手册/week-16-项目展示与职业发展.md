## 二、实验内容

### 步骤1：准备演示材料（30分钟）

#### 1.1 项目演示准备

准备以下演示内容：

1. **SQL查询演示用例**
   ```sql
   -- 创建表
   CREATE TABLE users (id INT, name TEXT, age INT);
   -- 插入数据
   INSERT INTO users VALUES (1, 'Alice', 25);
   -- 查询
   SELECT * FROM users WHERE age > 20;
   ```

2. **事务处理演示**
   ```sql
   BEGIN;
   UPDATE users SET age = 26 WHERE id = 1;
   COMMIT;
   ```

3. **性能测试演示**
   ```bash
   cargo bench
   ```

#### 1.2 代码演示准备

选择核心代码进行讲解：

- **解析器**：词法分析、语法分析
- **执行器**：查询执行流程
- **存储引擎**：页式存储、B+树

#### 1.3 文档整理

整理以下文档：

- 项目README
- API文档
- 设计文档

### 步骤3：问题回答（20分钟）

#### 3.1 准备技术问题

常见技术问题：

1. **为什么选择Rust？**
   - 内存安全
   - 性能接近C
   - 现代语言特性

2. **如何实现MVCC？**
   - 快照
   - 版本链
   - 可见性判断

3. **如何保证数据一致性？**
   - WAL日志
   - 两阶段提交
   - 锁机制

4. **如何优化查询性能？**
   - 索引
   - 查询计划优化
   - 缓存

5. **如何保证安全性？**

   - 错误处理
