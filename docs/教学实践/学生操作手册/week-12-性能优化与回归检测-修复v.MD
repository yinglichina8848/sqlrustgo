## 二、实验背景

### 什么是"回归"？

```
功能回归：昨天能跑的测试，今天跑不过了
性能回归：昨天 QPS 1000，今天 QPS 500
覆盖回归：昨天覆盖率 78%，今天 65%

最危险的回归 = 你不知道它发生了
→ 你的"优化"实际上"劣化"了
→ 直到上线后用户投诉，你才发现
```

### SQLRustGo 的回归检测标准

| 退化程度 | 判定 | 动作 |
|---------|------|------|
| ≤ 5% | PASS | 噪声范围内，正常 |
| 5% ~ 20% | WARN | PR 需要说明原因 |
| > 20% | FAIL | 必须修复，PR Blocked |

## 三、操作步骤

### 步骤1：运行回归检测（15分钟）

#### 1.1 运行 QPS 基准（作为"优化前"的基线）

```bash
# 记录当前性能数据
cargo test --test qps_benchmark_test -- --ignored --nocapture 2>&1 | grep "QPS:" > /tmp/before_qps.txt
cat /tmp/before_qps.txt
```

#### 1.2 检查回归检测脚本

```bash
# 查看项目自带的回归检测脚本
cat scripts/gate/check_regression.sh 2>/dev/null || echo "脚本不存在，我们将在下一步手动模拟"
```

#### 1.3 理解回归检测的逻辑

```
check_regression.sh 做了什么：
  1. 读取 current.json（上次记录的基准数据）
  2. 重新运行测试
  3. 对比：新数据 vs 基准数据
  4. 计算 Δ%（变化百分比）
  5. 判断：Δ ≤ 5% → PASS / 5% < Δ ≤ 20% → WARN / Δ > 20% → FAIL

关键：--skip-run 标志
  不实际运行测试，直接使用缓存数据
  → 结果永远是 Δ = 0%
  → 只能用于调试，不能用于 Gate 决策！
```

### 步骤3：理解自优化闭环 PDCA（20分钟）

#### 3.1 PDCA 循环

```
┌──────────────────────────────────────────┐
│          PDCA 自优化闭环                   │
├──────────────────────────────────────────┤
│                                          │
│  Plan（计划）                             │
│  "下个版本的目标：DELETE QPS ≥ 10,000"     │
│       │                                  │
│       ▼                                  │
│  Do（执行）                               │
│  实现优化 → 提交 PR #500                  │
│       │                                  │
│       ▼                                  │
│  Check（检查）                            │
│  Gate BP2: DELETE QPS = 950 → FAIL       │
│  回归检测: 对比优化前 → Δ = +360% ✅      │
│  但还是没达到 E-09 地板 → 需要继续优化    │
│       │                                  │
│       ▼                                  │
│  Act（改进）                              │
│  1. 分析根因：为什么还是慢？              │
│  2. 新一轮优化：批量提交 / 异步 WAL       │
│  3. 更新基准数据                          │
│  4. 沉淀知识到 GBrain                     │
│       │                                  │
│       └──→ 回到 Plan                     │
│                                          │
└──────────────────────────────────────────┘
```
