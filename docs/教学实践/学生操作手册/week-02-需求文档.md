# 第2周：结构化分析与设计实践

> 实验时间：2学时
> 实验类型：设计性
> 前置知识：第二讲 - 结构化分析、设计与实现

---

## 一、实验目标

- [ ] 掌握结构化分析的基本方法（DFD、ER图、数据字典）
- [ ] 掌握结构化设计的基本方法（模块结构图）
- [ ] 能够使用AI辅助生成各类图表
- [ ] 能够将分析设计转化为数据库建表语句

---

## 二、实验案例：图书管理系统

本实验以**图书管理系统**为案例，完成结构化分析和设计。

---

## 三、操作步骤

### 步骤1：系统需求分析（20分钟）

#### 1.1 确定系统功能

在AI对话窗口输入：

```
请为"图书管理系统"列出核心功能，包括：
1. 图书借阅功能
2. 图书归还功能
3. 图书查询功能
4. 读者管理功能
```

#### 1.2 确定用户角色

```
请列出图书管理系统的用户角色及其功能权限
```

#### ✅ 检查点1：记录系统功能和角色

---

### 步骤2：绘制DFD数据流图（25分钟）

#### 2.1 使用AI生成DFD上下文图

在Trae中输入：

```
请为"图书管理系统"生成数据流图（DFD）的上下文图，使用PlantUML语法。
外部实体：读者、图书管理员
系统：图书管理系统
数据存储：图书数据库、借阅记录数据库
请输出PlantUML代码。
```

#### 2.2 生成DFD Level 1

```
请为"图书管理系统"生成DFD的0层图（Level 1），展示主要处理功能：
- 借书处理
- 还书处理
- 图书查询
- 读者管理
```

#### 2.3 查看生成的图表

将AI生成的PlantUML代码复制到支持PlantUML预览的编辑器中查看效果。

#### ✅ 检查点2：保存DFD图表

---

### 步骤3：绘制ER图（25分钟）

#### 3.1 分析实体和关系

```
请为"图书管理系统"分析：
1. 需要哪些实体（读者、图书、借阅记录等）
2. 每个实体的属性
3. 实体之间的关系（1:1, 1:N, M:N）
```

#### 3.2 使用AI生成ER图

```
请为"图书管理系统"生成ER图，使用PlantUML语法。
实体：
- 读者（读者编号、姓名、电话、借阅状态）
- 图书（ISBN、书名、作者、库存）
- 借阅记录（记录编号、借阅日期、应还日期、实际归还日期）
关系：
- 读者 1:N 借阅记录
- 图书 1:N 借阅记录
请输出PlantUML代码。
```

#### ✅ 检查点3：保存ER图

---

### 步骤4：创建数据字典（15分钟）

#### 4.1 分析数据存储

根据ER图，列出所有数据表及字段：

| 表名 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 读者 | ... | ... | ... |
| 图书 | ... | ... | ... |
| 借阅记录 | ... | ... | ... |

#### 4.2 使用AI生成数据字典

```
请为"图书管理系统"创建数据字典，包含：
1. 读者表字段定义
2. 图书表字段定义
3. 借阅记录表字段定义
格式：字段名、类型、说明、主键/外键
```

#### ✅ 检查点4：保存数据字典

---

### 步骤5：设计模块结构图（20分钟）

#### 5.1 分析系统模块

```
请分析图书管理系统应该有哪些模块：
- 前端交互模块
- 业务逻辑模块
- 数据访问模块
```

#### 5.2 使用AI生成模块结构图

```
请为"图书管理系统"生成模块结构图，使用PlantUML语法。
模块包括：
- 借阅管理模块
- 还书管理模块
- 图书查询模块
- 读者管理模块
- 数据库访问层
请展示模块之间的调用关系。
```

#### ✅ 检查点5：保存模块结构图

---

### 步骤6：生成SQL建表语句（20分钟）

#### 6.1 使用AI生成SQL

```
请根据以下ER图生成MySQL建表SQL语句：

读者表(READER)：
- reader_id: 读者ID (INT, PK)
- name: 姓名 (VARCHAR)
- phone: 电话 (VARCHAR)
- register_date: 注册日期 (DATE)

图书表(BOOK)：
- book_id: 图书ID (INT, PK)
- isbn: ISBN (VARCHAR)
- title: 书名 (VARCHAR)
- author: 作者 (VARCHAR)
- stock: 库存 (INT)

借阅记录表(BORROW)：
- borrow_id: 借阅记录ID (INT, PK)
- reader_id: 读者ID (INT, FK)
- book_id: 图书ID (INT, FK)
- borrow_date: 借阅日期 (DATE)
- return_date: 应还日期 (DATE)
- actual_return_date: 实际归还日期 (DATE, NULLABLE)

关系：
- 读者 1:N 借阅记录
- 图书 1:N 借阅记录

请生成完整的CREATE TABLE语句，包含主键和外键约束。
```

#### 6.2 验证SQL语法

将生成的SQL在数据库中执行，验证语法正确性。

#### ✅ 检查点6：保存SQL建表语句
