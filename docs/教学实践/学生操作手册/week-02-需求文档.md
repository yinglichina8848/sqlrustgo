## 二、操作步骤

### 步骤1：使用AI分析需求（25分钟）

#### 1.1 打开TRAE IDE

启动TRAE IDE，打开SQLRustGo项目。

#### 1.2 输入需求描述

在AI对话窗口输入：

```
请分析SQLRustGo项目的需求，包括：
1. 核心功能需求
2. 非功能需求
3. 用户角色
4. 使用场景
```

#### 1.3 分析AI输出

记录AI给出的需求分析结果，思考是否合理。

#### ✅ 检查点1：保存AI分析结果

---

### 步骤2：编写需求规格说明书（30分钟）

#### 2.1 创建文档目录

```bash
mkdir -p docs/requirements
```

#### 2.2 创建需求文档

创建文件 `docs/requirements/requirements.md`：

```markdown
# 需求规格说明书

## 1. 项目背景

SQLRustGo 是一个用 Rust 实现的 SQL 数据库系统...

## 2. 功能需求

### 2.1 SQL解析功能
- 支持SELECT语句
- 支持INSERT语句
- 支持UPDATE语句
- 支持DELETE语句

### 2.2 查询执行功能
- 支持WHERE条件过滤
- 支持ORDER BY排序
- 支持LIMIT分页

### 2.3 数据存储功能
- 支持表结构定义
- 支持数据持久化
- 支持索引

## 3. 非功能需求

### 3.1 性能要求
- 查询响应时间 < 100ms
- 支持10万行数据

### 3.2 可靠性要求
- 数据不丢失
- 事务ACID

### 3.3 可扩展性要求
- 模块化设计
- 插件化架构

## 4. 需求优先级

| 需求 | 优先级 | 说明 |
|------|--------|------|
| SELECT | P0 | 核心功能 |
| INSERT | P0 | 核心功能 |
| UPDATE | P1 | 重要功能 |
| DELETE | P1 | 重要功能 |

## 5. 验收标准

- 所有P0需求实现
- 测试覆盖率 ≥ 80%
- 文档完整
```

#### ✅ 检查点2：保存需求文档

---

### 步骤3：创建用户故事（20分钟）

#### 3.1 创建用户故事文件

创建文件 `docs/requirements/user-stories.md`：

```markdown
# 用户故事

## US-001: 执行SELECT查询

作为 用户
我想要 执行SELECT查询语句
以便于 从数据库中检索数据

验收标准：
- 支持基本的SELECT语句
- 支持WHERE条件
- 支持ORDER BY排序

## US-002: 执行INSERT操作

作为 用户
我想要 执行INSERT语句插入数据
以便于 向数据库中添加新记录

验收标准：
- 支持单行插入
- 支持多行插入
- 返回插入行数

## US-003: 执行UPDATE操作

作为 用户
我想要 执行UPDATE语句更新数据
以便于 修改数据库中的记录

验收标准：
- 支持WHERE条件
- 返回更新行数

## US-004: 执行DELETE操作

作为 用户
我想要 执行DELETE语句删除数据
以便于 从数据库中删除记录

验收标准：
- 支持WHERE条件
- 返回删除行数
```

#### ✅ 检查点3：保存用户故事

### 步骤4：Git提交（15分钟）

#### 4.1 创建分支

```bash
git checkout -b docs/requirements-你的学号
```

#### 4.2 添加文件

```bash
git add docs/requirements/
```

#### 4.3 提交

```bash
git commit -m "docs: add requirements document"
```

#### 4.4 推送

```bash
git push origin docs/requirements-你的学号
```
