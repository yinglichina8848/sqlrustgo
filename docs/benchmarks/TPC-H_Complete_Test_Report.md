## 扩展性分析

### SF=1 vs SF=10 性能增长倍数

| 数据库 | SF=1 平均 | SF=10 平均 | 增长倍数 |
|--------|-----------|------------|----------|
| SQLRustGo | 216.0 ms | 4879.5 ms | **22.6x** |
| SQLite | 68.2 ms | 1147.0 ms | 16.8x |
| PostgreSQL | 289.6 ms | 759.6 ms | **2.6x** |

**结论**：
- PostgreSQL 扩展性最好（2.6x），接近线性扩展
- SQLite 扩展性中等（16.8x），部分查询在 SF=10 下退化
- SQLRustGo 扩展性较差（22.6x），固定开销占比大

## SQLRustGo 性能分析

### SF=1 vs SF=10 对比

| 指标 | SF=1 | SF=10 | 变化 |
|------|------|-------|------|
| 平均耗时 | 216.0 ms | 4879.5 ms | +22.6x |
| 最短查询 | 205.0 ms | 4258.0 ms | +20.8x |
| 最长查询 | 251.0 ms | 6277.0 ms | +25.0x |
| 性能波动 | 46 ms | 2019 ms | +43.9x |

### 问题诊断

1. **查询时间与数据规模非线性关系**
   - SF=1 → SF=10（10x 数据），但性能下降 22.6x
   - 说明存在 O(n²) 或更差的算法复杂度

2. **无查询优化**
   - 所有查询耗时相近（4258-6277ms），无视复杂度
   - 简单查询（Q2, Q9）和复杂查询（Q7, Q11）耗时相同

3. **固定开销大**
   - 执行引擎初始化、内存分配等固定开销约 4-5 秒

### SQLRustGo vs 竞品

| 规模 | vs SQLite | vs PostgreSQL |
|------|-----------|---------------|
| SF=1 | 慢 3.2x | 快 0.7x |
| SF=10 | 慢 4.3x | 慢 6.4x |

**结论**：SF=1 时 SQLRustGo 与竞品差距较小，SF=10 时差距明显扩大

## 结论与建议

### 性能排名（综合）

| 排名 | SF=1 | SF=10 |
|------|------|-------|
| 1 | SQLite | PostgreSQL |
| 2 | SQLRustGo | SQLite |
| 3 | PostgreSQL | SQLRustGo |

### 优化建议

#### SQLRustGo
1. 添加查询计划器和成本估算
2. 实现索引支持（B+ Tree）
3. 优化内存管理减少固定开销
4. 添加结果缓存

#### PostgreSQL
1. 扩展性最优，继续保持
2. 复杂查询（Q3, Q18）可进一步优化

#### SQLite
1. 简单查询性能极佳
2. 复杂查询（Q5, Q11）需优化 JOIN 策略
