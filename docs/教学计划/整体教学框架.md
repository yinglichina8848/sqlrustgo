# AI增强的软件工程 - 整体教学框架

## 课程设计理念

### 核心思想

**以项目为主线，以实践为驱动，以AI为增强**

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        课程设计核心理念                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   传统软件工程教学：                                                          │
│   理论 ──→ 作业 ──→ 考试                                                     │
│   问题：理论与实践脱节，学生难以理解抽象概念                                   │
│                                                                              │
│   本课程设计：                                                                │
│   项目需求 ──→ 理论讲解 ──→ AI辅助实践 ──→ 项目迭代 ──→ 总结提升              │
│   优势：以真实项目为载体，理论指导实践，实践验证理论                            │
│                                                                              │
│   v3版本设计理念：                                                            │
│   上半学期：单一AI工具 + UML建模 + 快速原型（降低难度）                        │
│   下半学期：协同开发 + 工程治理 + 发布流程（提升能力）                          │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 项目主线：SQLRustGo

SQLRustGo 是一个用 Rust 实现的 SQL 数据库内核，贯穿整个学期的教学：

| 阶段 | 项目版本 | 教学重点 |
|------|----------|----------|
| 第1-4周 | 草稿版本 | UML建模、结构化设计、面向对象设计、快速原型法 |
| 第5-8周 | Alpha版本 | AI辅助编码、测试驱动开发、核心模块实现 |
| 第9-12周 | Beta版本 | Git分支策略、PR流程、CI/CD、用户文档 |
| 第13-16周 | RC1/Release | 安全扫描、发布门禁、版本发布、项目演示 |
