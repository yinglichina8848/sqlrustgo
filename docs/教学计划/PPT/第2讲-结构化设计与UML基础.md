# Part 1: 结构化设计方法

---

## 1.1 结构化设计核心思想

### 自顶向下、逐步求精

```
系统
  │
  ├── 子系统1
  │    ├── 模块1.1
  │    └── 模块1.2
  │
  └── 子系统2
       ├── 模块2.1
       └── 模块2.2
```

### 设计原则

- **高内聚**：模块只做一件事
- **低耦合**：模块之间依赖少
- **信息隐藏**：模块内部细节隐藏

---

## 1.2 数据流图（DFD）

### 符号

```
○ → 加工（Process）
□ → 数据源（Source/Sink）
→ → 数据流（Data Flow）
~ → 数据存储（Data Store）
```

# Part 4: AI辅助UML绘制

---

## 4.1 使用AI生成PlantUML

### 提示词示例

```
帮我生成SQLRustGo的用例图PlantUML代码：
- 参与者：用户、管理员
- 用例：执行查询、管理表、管理用户
- 包含关系
```

### AI生成结果

```plantuml
@startuml
left to right direction
actor User as "用户"
actor Admin as "管理员"
rectangle "SQLRustGo" {
  User --> (执行SQL查询)
  User --> (管理表)
  Admin --> (管理用户)
}
@enduml
```

---

## 4.2 自动档阶段的AI使用技巧

### 正确方式

| 场景 | AI能做的 | 人类要做的 |
|------|---------|-----------|
| 理解概念 | 解释定义 | 判断是否正确 |
| 绘制图形 | 生成代码 | 检查逻辑 |
| 改进设计 | 提供建议 | 做出决策 |

### 关键原则

- **AI生成** ≠ **最终方案**
- **必须理解** ≠ **直接使用**
- **参考为主** ≠ **复制为准**
