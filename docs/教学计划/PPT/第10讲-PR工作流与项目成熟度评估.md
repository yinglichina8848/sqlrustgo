## 代码审核的艺术

### 审核者的职责

| 职责 | 说明 |
|------|------|
| 功能正确性 | 代码是否实现了预期功能 |
| 代码质量 | 是否清晰、可读、可维护 |
| 安全性 | 是否有安全漏洞 |
| 性能 | 是否有性能问题 |
| 测试 | 是否有足够的测试覆盖 |

### 审核意见类型

- **MUST** - 必须修改（阻塞合并）
- **SHOULD** - 建议修改（推荐但非强制）
- **NICE TO HAVE** - 可选改进
- **QUESTION** - 问题讨论
