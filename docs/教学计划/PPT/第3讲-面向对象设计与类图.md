# Part 1: 面向对象设计原则

---

## 1.1 核心概念

### 封装

- 把数据和操作放在一起
- 对外隐藏实现细节

### 继承

- 子类复用父类的属性和行为
- 建立"is-a"关系

### 多态

- 同一接口，不同实现
- 提高代码灵活性

---

## 1.2 SOLID原则

| 原则 | 全称 | 说明 |
|------|------|------|
| SRP | 单一职责 | 一个类只做一件事 |
| OCP | 开闭原则 | 对扩展开放，对修改关闭 |
| LSP | 里氏替换 | 子类可以替换父类 |
| ISP | 接口隔离 | 接口要小而专 |
| DIP | 依赖倒置 | 依赖抽象，不依赖具体 |

# Part 2: 类图

---

## 2.1 类图元素

```
┌─────────────────────────────────┐
│           类名                   │
├─────────────────────────────────┤
│ - 属性 : 类型                   │
│ - 属性 : 类型                   │
├─────────────────────────────────┤
│ + 方法() : 返回类型            │
│ # 方法() : 返回类型            │
└─────────────────────────────────┘

- private  # protected  + public
```

---

## 2.2 类之间的关系

| 关系 | 符号 | 说明 |
|------|------|------|
| 关联 | → | 一般关系 |
| 聚合 | ◇─ | 部分-整体，可分离 |
| 组合 | ◆─ | 部分-整体，不可分离 |
| 依赖 | ..> | 临时使用 |
| 泛化 | △ | 继承 |
