# Part 1: 顺序图

---

## 1.1 顺序图元素

```
┌────────┐     ┌────────┐     ┌────────┐
│  对象1 │     │  对象2 │     │  对象3 │
└────┬───┘     └────┬───┘     └────┬───┘
     │                │                │
     │──── 消息1 ────>│                │
     │                │──── 消息2 ────>│
     │<─── 返回 ─────│                │
     │                │                │
     ▼                ▼                ▼
  生命线           生命线           生命线
```

---

## 1.2 SQLRustGo顺序图示例

```
用户 ──> REPL: 输入SQL
REPL ──> Parser: 解析SQL
Parser ──> Lexer: 词法分析
Lexer ──> Parser: 返回Token
Parser ──> Executor: 生成执行计划
Executor ──> Storage: 读取数据
Storage ──> Executor: 返回数据
Executor ──> REPL: 返回结果
REPL ──> 用户: 显示结果
```

# Part 3: 架构图

---

## 3.1 SQLRustGo四层架构

```
┌─────────────────────────────────────────┐
│           Parser Layer                  │
│    ┌───────┐  ┌───────┐  ┌────────┐ │
│    │ Lexer │→ │Parser │→ │  AST   │ │
│    └───────┘  └───────┘  └────────┘ │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│           Planner Layer                 │
│    ┌───────────┐  ┌─────────────┐     │
│    │LogicalPlan│→ │PhysicalPlan│     │
│    └───────────┘  └─────────────┘     │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│           Executor Layer               │
│    ┌────────┐  ┌────────┐  ┌───────┐│
│    │ Select │  │ Insert │  │ Update│ │
│    └────────┘  └────────┘  └───────┘│
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│           Storage Layer                │
│  ┌─────────┐ ┌──────────┐ ┌────────┐  │
│  │  Page  │ │BufferPool│ │ B+Tree│  │
│  └─────────┘ └──────────┘ └────────┘  │
└─────────────────────────────────────────┘
```
