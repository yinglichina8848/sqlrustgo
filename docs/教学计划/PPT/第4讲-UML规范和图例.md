# Part 1: UML简介与分类

---

## 1.1 UML是什么？

### 统一建模语言（Unified Modeling Language）

- 软件工程的标准建模语言
- 可视化、规范化、文档化
- 不是方法论，而是工具

### UML图分类

| 类别 | 图类型 |
|------|--------|
| 结构图 | 类图、对象图、包图、组件图、部署图 |
| 行为图 | 用例图、活动图、状态图 |
| 交互图 | 顺序图、通信图、时间图 |

---

## 1.2 UML在AI时代的价值

- **沟通桥梁**：人机对话的标准化语言
- **AI生成**：自然语言 → UML
- **代码同步**：UML ↔ 代码

# Part 2: 结构图

---

## 2.1 类图（Class Diagram）

### 类的表示

```
┌─────────────────────────┐
│        类名             │
├─────────────────────────┤
│ - 属性1: 类型           │
│ - 属性2: 类型           │
├─────────────────────────┤
│ + 方法1(): 返回类型     │
│ + 方法2(参数): 返回类型 │
└─────────────────────────┘
```

### 可见性符号

| 符号 | 含义 |
|------|------|
| + | public |
| - | private |
| # | protected |
| ~ | package |

---

## 2.2 类图关系

| 关系 | 符号 | 示例 |
|------|------|------|
| 继承 | ──▶ (空心三角) | `A ──▶ B` |
| 实现 | ──▶ (虚线空心三角) | `A -.▶ B` |
| 关联 | ──→ | `A ──→ B` |
| 依赖 | ──→ (虚线) | `A .─→ B` |
| 聚合 | ◇── | `A ◇── B` |
| 组合 | ◆── | `A ◆── B` |

---

## 2.3 对象图（Object Diagram）

- 类的实例化
- 某一时刻的对象快照
- 展示对象之间的关系

---

## 2.4 组件图（Component Diagram）

### 用途

- 描述系统的物理结构
- 展示组件及其依赖
- 接口和连接件

---

## 2.5 部署图（Deployment Diagram）

### 用途

- 描述系统物理部署
- 节点、组件、网络

---

# Part 3: 行为图

---

## 3.1 用例图（Use Case Diagram）

### 元素

- **参与者**：外部交互者
- **用例**：系统功能
- **系统边界**：系统范围

### 关系

| 关系 | 符号 |
|------|------|
| 包含 | `..>` (<<include>>) |
| 扩展 | `..>` (<<extend>>) |
| 继承 | `─▶` |

---

## 3.2 活动图（Activity Diagram）

### 基本元素

| 元素 | 符号 |
|------|------|
| 活动 | 圆角矩形 |
| 开始 | ● |
| 结束 | ◎ |
| 分支 | 菱形 |
| 并发 | 粗横线 |

---

## 3.3 状态图（State Diagram）

### 元素

- **状态**：圆角矩形
- **初始状态**：●
- **终止状态**：◎
- **转换**：箭头 + 事件

### 事务状态示例

```
[空闲] ──BEGIN──▶ [活跃]
[活跃] ──COMMIT──▶ [已提交]
[活跃] ──ROLLBACK──▶ [已回滚]
```

---

# Part 4: 交互图

---

## 4.1 顺序图（Sequence Diagram）

### 元素

| 元素 | 说明 |
|------|------|
| 生命线 | 垂直虚线 |
| 激活条 | 细长矩形 |
| 消息 | 箭头 |

### 消息类型

```
→   同步消息
─>>  异步消息
←   返回消息
```

---

## 4.2 通信图（Communication Diagram）

- 展示对象之间的关系
- 消息编号表示顺序

# Part 5: AI辅助UML实践

---

## 5.1 提示词技巧

### 有效提示词

```
请用PlantUML生成[图类型]，描述[场景]。
参与者：[...]
步骤：[...]
```

### 示例

```
请为SQL查询生成顺序图：
参与者：Client、Parser、Executor、Storage
1. Client发送SQL
2. Parser解析
3. Executor执行
4. Storage返回结果
```
