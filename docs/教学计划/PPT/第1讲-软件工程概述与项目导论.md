# Part 1: 软件工程发展历史

---

## 1.1 软件工程的诞生

### 1968年 NATO会议

- **"软件危机"的提出**
  - 项目延期、预算超支、质量低下
  - 复杂度指数级增长

### 解决方案：软件工程

- 规范化、流程化、工具化
- 从"个人艺术"到"工业生产"

# Part 2: AI对软件工程的影响

---

## 2.1 AI改变编程方式

### 传统编程 vs AI编程

| 方面 | 传统编程 | AI编程 |
|------|----------|--------|
| 方式 | 人类写代码 | 人类描述，AI生成 |
| 效率 | 10行/小时 | 100行/小时 |
| 门槛 | 高 | 低 |
| 质量 | 依赖个人 | 依赖提示词 |

## 2.3 AI编程的能力边界

### AI擅长

- 代码生成（模板代码）
- 解释概念
- Bug定位
- 测试生成

### AI不擅长

- 理解真实需求
- 架构决策
- 业务逻辑判断
- 异常情况处理

# Part 3: 🚗 自动档开发模式

---

## 3.1 什么是"自动档"？

### 比喻

```
┌─────────────────────────────────────────────────────────────────────┐
│                    自动档 vs 手动档                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  🚗 自动档（AI辅助开发）                                          │
│  ════════════════════════════                                       │
│  • 驾驶员（人类）说目标                                            │
│  • 自动档（AI）执行                                               │
│  • 人类只需关注"去哪"                                             │
│  • 不需要掌握驾驶技术                                              │
│                                                                      │
│  🕹️ 手动档（传统开发）                                            │
│  ════════════════════════════                                       │
│  • 驾驶员（人类）亲自操作                                          │
│  • 每一步都需要人类执行                                            │
│  • 需要掌握驾驶技术                                                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

---

## 3.2 自动档的优势

### 降低门槛

- 不需要精通编程语言
- 不需要理解底层原理
- 快速看到成果

### 提高效率

- AI生成基础代码
- 人类只需修改和审查
- 快速迭代

### 适合学习

- 先"会用"再"懂原理"
- 通过结果理解过程
- 获得成就感

---

## 3.3 自动档的局限

### 不能完全依赖AI

```
┌─────────────────────────────────────────────────────────────────────┐
│                    自动档的局限                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  😱 问题1：AI生成的代码可能有bug                                   │
│  ├── AI不理解你的真正需求                                          │
│  ├── AI生成的代码可能不完整                                        │
│  └── AI不知道边界情况                                              │
│                                                                      │
│  😱 问题2：不会真正"学会"                                          │
│  ├── 只知道"怎么用"不知道"为什么"                                  │
│  ├── 离开AI寸步难行                                                │
│  └── 无法独立解决问题                                              │
│                                                                      │
│  😱 问题3：复杂任务处理不了                                        │
│  ├── 架构设计                                                     │
│  ├── 性能优化                                                     │
│  └── 系统调试                                                     │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

---

## 3.4 自动档的正确使用方式

### 核心原则

| 原则 | 说明 |
|------|------|
| AI是辅助 | 人类主导，AI执行 |
| 理解优先 | 不只看结果，要理解过程 |
| 审查必要 | AI的输出必须人工审查 |
| 渐进学习 | 从"会用"到"懂原理" |
