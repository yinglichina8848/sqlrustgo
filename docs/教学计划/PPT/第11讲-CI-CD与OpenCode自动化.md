## GitHub Actions核心概念

### 工作流（Workflow）

- 自动化流程的定义
- 触发条件：push、PR、定时等

### 作业（Job）

- 一组步骤的集合
- 可并行执行

### 步骤（Step）

- 具体的动作
- 可以使用action或直接运行命令

### 示例：SQLRustGo CI配置

```yaml
name: CI

on:
  push:
    branches: [ develop/v2.6.0 ]
  pull_request:
    branches: [ develop/v2.6.0 ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: cargo build --all-features
      - run: cargo test --all-features
      - run: cargo clippy --all-features -- -D warnings
      - run: cargo fmt --check --all
```

## 多AI协作流程设计

### AI角色定义

| 角色 | 工具 | 任务 |
|------|------|------|
| Analyst | Claude | 需求分析、编写需求文档 |
| Architect | GPT-4 | 架构设计、技术选型 |
| Developer | Claude Code | 编码实现、单元测试 |
| Tester | Claude | 集成测试、性能测试 |
| Reviewer | Claude | 代码审查、质量把关 |

### 协作流程设计原则

1. **串行为主**：需求→设计→实现→测试→审查
2. **明确依赖**：每阶段依赖前一阶段完成
3. **角色隔离**：不同AI负责不同阶段
4. **人工监督**：关键节点需要人工确认
