# Part 2: 版本号管理

---

## 2.1 语义化版本

### 格式

```
MAJOR.MINOR.PATCH

示例：
1.0.0 → 1.0.1 (patch)  // Bug修复
1.0.1 → 1.1.0 (minor)  // 新功能
1.1.0 → 2.0.0 (major) // 重大变更
```

---

## 2.2 Git标签

```bash
# 创建标签
git tag -a v1.0.0 -m "Release v1.0.0"

# 推送标签
git push origin v1.0.0

# 查看标签
git tag -l
```
