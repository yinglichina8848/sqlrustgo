# Part 1: 多AI时代的安全挑战

---

## 1.1 AI生成代码的安全风险

### AI不知道"安全"

```
┌─────────────────────────────────────────────────────────────────────┐
│                    AI的安全盲区                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  😱 问题1：SQL注入                                               │
│  AI: "帮我写一个查询用户的函数"                                    │
│  AI生成:                                                          │
│  ```rust                                                          │
│  fn query_user(name: &str) {                                      │
│      let sql = format!("SELECT * FROM users WHERE name = '{}'",   │
│                        name);  // ❌ 危险！                       │
│  }                                                                 │
│  ```                                                                │
│                                                                      │
│  😱 问题2：硬编码密码                                             │
│  AI: "帮我连接数据库"                                             │
│  AI生成:                                                           │
│  ```rust                                                           │
│  let password = "admin123";  // ❌ 危险！                        │
│  ```                                                                │
│                                                                      │
│  😱 问题3：不安全的加密                                           │
│  AI: "帮我加密密码"                                               │
│  AI生成:                                                           │
│  ```rust                                                           │
│  let hash = md5(password);  // ❌ 危险！MD5不安全              │
│  ```                                                                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 1.2 多AI协作的安全问题

### 多个AI = 多个风险点

```
┌─────────────────────────────────────────────────────────────────────┐
│                    多AI的安全风险                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  • AI #1 (Parser) 可能引入SQL注入                                  │
│  • AI #2 (Executor) 可能引入命令注入                              │
│  • AI #3 (Storage) 可能引入路径遍历                              │
│  • AI #4 (Network) 可能引入协议漏洞                              │
│                                                                      │
│  问题：                                                            │
│  • 不知道是哪个AI引入的问题                                        │
│  • 多个AI的代码集成后可能产生新漏洞                               │
│  • 缺乏整体安全视角                                               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

---

## 1.3 解决方案：安全治理

### 安全门禁

```
┌─────────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────────┤

│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 2.3 Dependabot配置

### 自动依赖更新

```yaml
# .github/dependabot.yml
version: 2

updates:
  - package-ecosystem: "cargo"
    directory: "/"
    schedule:
      interval: "weekly"
    open-pull-requests-limit: 10
    
  - package-ecosystem: "github-actions"
    directory: "/"
    schedule:
      interval: "weekly"
```

# Part 3: 代码扫描

---

## 3.1 代码安全扫描

### 常见安全问题

| 问题 | 危险等级 | 示例 |
|------|---------|------|
| SQL注入 | 高 | 字符串拼接SQL |
| 命令注入 | 高 | 执行系统命令 |
| 路径遍历 | 高 | 未验证的文件路径 |
| 硬编码密码 | 中 | 代码中写密码 |
| 不安全加密 | 中 | 使用MD5 |
| 信息泄露 | 中 | 打印敏感信息 |

---

## 3.2 扫描工具

### Clippy安全检查

```bash
# 运行所有检查
cargo clippy -- -W clippy::all
```

### CodeQL

```yaml
# .github/workflows/codeql.yml
name: CodeQL

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: github/codeql-action/init@v2
        with:
          languages: rust
      - run: cargo build
      - uses: github/codeql-action/analyze@v2
```

## 3.3 安全审查清单

### AI代码必查项

```markdown
## 安全审查清单

### 输入验证
- [ ] 所有用户输入都验证了吗？

```

---

## 4.2 安全代码审查

```rust
// 不安全的代码示例 ❌
fn query(name: String) -> Result<User> {
    let sql = format!("SELECT * FROM users WHERE name = '{}'", name);
    execute(&sql)
}

// 安全代码示例 ✅
fn query(name: &str) -> Result<User> {
    let sql = "SELECT * FROM users WHERE name = ?";
    execute_with_params(sql, &[name])
}
```

---

## 4.3 配置Dependabot

```bash
# 创建配置文件

version: 2
updates:
  - package-ecosystem: "cargo"
    directory: "/"
    schedule:
      interval: "weekly"
EOF

# 提交
git add .
git commit -m "ci: add Dependabot config"
git push origin develop/v2.6.0
```

# 核心知识点总结

---

## 1. AI时代的安全挑战

- **AI不懂安全**：生成代码可能包含漏洞
- **多AI风险**：多个AI = 多个风险点
- **供应链**：依赖漏洞防不胜防

## 2. 安全门禁

- **依赖扫描**：cargo audit、Dependabot
- **代码扫描**：Clippy、CodeQL
- **人工审查**：业务逻辑检查

## 3. 安全实践

- **预防为主**：输入验证、最小权限
- **检测为辅**：自动化扫描、持续监控
- **响应快速**：漏洞修复、安全更新
