# Part 1: 架构设计概述

---

## 1.1 什么是软件架构

### 定义

软件系统的高层结构，包括：
- 组件（Components）
- 连接器（Connectors）
- 约束（Constraints）

### 为什么需要架构

- 复杂性管理
- 质量保证
- 团队协作
- 风险控制

---

## 1.2 常见架构风格

| 架构 | 特点 | 适用场景 |
|------|------|----------|
| 分层架构 | 清晰分离 | 通用 |
| 微服务 | 独立部署 | 大型系统 |
| 事件驱动 | 异步通信 | 实时系统 |
| MVC | 分离关注点 | Web应用 |

# Part 2: SQLRustGo四层架构

---

## 2.1 分层架构

```
┌───────────────────────────────────────────────┐
│              Parser Layer                    │
│  Lexer → Parser → AST                      │
├───────────────────────────────────────────────┤
│              Planner Layer                   │
│  LogicalPlan → PhysicalPlan                 │
├───────────────────────────────────────────────┤
│              Executor Layer                  │
│  Volcano Model → Operators                  │
├───────────────────────────────────────────────┤
│              Storage Layer                  │
│  Page → BufferPool → B+Tree → WAL         │
└───────────────────────────────────────────────┘
```

---

## 2.2 每层职责

| 层 | 职责 | 核心组件 |
|---|------|----------|
| Parser | SQL解析 | Lexer, Parser |
| Planner | 查询计划 | Optimizer |
| Executor | 执行查询 | Operators |
| Storage | 数据存储 | BufferPool, B+Tree |
