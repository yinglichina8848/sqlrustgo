# Part 2: 性能分析方法

---

## 2.1 性能分析工具

### Rust性能工具

| 工具 | 作用 | 使用场景 |
|------|------|---------|
| cargo bench | 基准测试 | 测量性能 |
| cargo flamegraph | 火焰图 | 找热点 |
| perf | Linux性能分析 | 系统级分析 |
| cargo-bloat | 代码体积分析 | 找大函数 |

---

## 2.2 性能分析流程

### 四步分析法

```
1. 建立基准
   ├── 测量当前性能
   └── 设定性能目标

2. 识别瓶颈
   ├── 使用profiler
   ├── 定位热点代码
   └── 分析调用栈

3. 优化代码
   ├── 算法优化
   ├── 减少分配
   └── 并行化

4. 验证效果
   ├── 对比优化前后
   └── 确认达到目标
```

# Part 3: 代码重构

---

## 3.1 重构的原则

### 何时重构

| 情况 | 说明 | 多AI注意事项 |
|------|------|-------------|
| 重复代码 | 同一逻辑出现多次 | 确保所有AI知道新位置 |
| 过长函数 | 超过50行 | AI容易写出长函数 |
| 命名不当 | 变量名无意义 | AI英文命名可能奇怪 |
| 耦合严重 | 模块间强依赖 | AI容易忽略接口 |

---

## 3.2 重构的安全实践

### 重构检查清单

```markdown
## 重构前
- [ ] 理解现有代码逻辑
- [ ] 确保有测试覆盖
- [ ] 通知相关AI模块

## 重构中
- [ ] 小步提交
- [ ] 保持功能不变
- [ ] 更新依赖方

## 重构后
- [ ] 运行所有测试
- [ ] 运行性能基准
- [ ] 通知相关AI模块
```

# Part 4: 实践练习

---

## 4.1 性能分析实践

```bash
# 安装工具
cargo install cargo-bench

# 运行基准测试
cargo bench

# 生成火焰图
cargo install cargo-flamegraph
cargo flamegraph --bin sqlrustgo
```

---

## 4.2 重构实践

```bash
# 1. 确保测试通过
cargo test

# 2. 小步重构
git checkout -b refactor/optimize-lexer

# 3. 重构代码
# ...

# 4. 验证
cargo test
cargo clippy

# 5. 提交PR
git add .
git commit -m "refactor: optimize lexer performance"
git push origin refactor/optimize-lexer
```

# 核心知识点总结

---

## 1. 多AI性能挑战

- **重复计算**：AI之间可能重复实现
- **低效算法**：AI不理解业务场景
- **内存问题**：AI容易忽略资源管理

## 2. 性能分析

- **工具**：bench、flamegraph
- **流程**：基准 → 瓶颈 → 优化 → 验证

## 3. 重构

- **安全重构**：测试保护、小步提交
- **多AI协作**：通知相关模块
