# Project Vision

> 项目愿景与战略定位

---

## 核心原则

- **稳定优先于功能**
- **纪律优先于速度**
- **发布优先于重构**

---

## 版本战略

| 版本 | 目标 | 定位 |
|------|------|------|
| v1.x | 稳定、规范、可维护 | 工程合规版本 |
| v2.x | 性能跃迁、架构升级 | 能力跃迁版本 |
| v3.x | 分布式执行平台 | 系统能力扩展 |
| v4.x | AI驱动优化 | 智能化转型 |

---

## v1.x 目标

- 可运行
- 不崩溃
- 有测试
- 有文档
- 有版本
- 有 CI
- 有 Release Note
- 符合工程规范

**不是**：更快、更复杂、更多 SQL、更高级优化器

---

## v2.x 目标

- 向量化执行
- CBO 优化器
- 并行执行
- 存储抽象层

---

## 长期愿景

构建一个可持续演进的数据库内核框架：

1. **执行模型抽象清晰**
2. **优化器独立模块**
3. **存储可替换**
4. **错误模型统一**
5. **性能可测量**

---

## 战略分离

```
1.x = 稳定
2.x = 技术跃迁
3.x = 系统能力
```

**不要跨阶段。不要在 1.x 做架构革命。**
