# v2.8.0 发布流程 (Release Process)

> **版本**: 1.0
> **更新日期**: 2026-05-01
> **维护人**: macmini opencode
> **适用版本**: v2.8.0+

---

## 一、发布阶段概览

v2.8.0 采用标准的发布生命周期：

```
Draft → Alpha → Beta → RC → GA
  │        │       │     │    │
  │        │       │     │    └── 正式发布
  │        │       │     └── 候选发布
  │        │       └── 功能冻结
  │        └── 持续集成
  └── 开发中
```

### 1.1 阶段定义

| 阶段 | 说明 | 持续时间 | 门禁 |
|------|------|----------|------|
| Draft | 设计与开发 | 1-2 周 | 无 |
| Alpha | 功能开发 | 2-3 周 | A-Gate |
| Beta | 功能冻结 | 1-2 周 | B-Gate |
| RC | 候选发布 | 1 周 | R-Gate |
| GA | 正式发布 | - | G-Gate |

      │       │       │       ▼

      │       │       │       │
      │       │       │       ├── G-Gate 通过
      │       │       │       │       │

      │       │       │
      │       │       └── 合并回 develop
      │       │
      └── 合并回 develop (热修复除外)
```

## 三、各阶段详细流程

### 3.1 Draft 阶段

**目标**: 完成架构设计和核心功能

**活动**:
- 架构设计评审
- 核心模块开发
- 技术验证 (POC)

**产出**:
- 设计文档
- 原型代码
- 技术方案

**入口条件**: 无
**出口条件**: 架构设计评审通过

---

### 3.2 Alpha 阶段

**目标**: 完成计划的功能开发

**活动**:
- 功能开发
- 单元测试编写
- 持续集成配置
- A-Gate 检查

**Tag 示例**:
```
v2.8.0-alpha1
v2.8.0-alpha2
```

**产出**:
- 功能代码
- 测试用例
- CI 配置

**入口条件**: A-Gate 通过
**出口条件**:
- 所有计划功能已实现
- 覆盖率 ≥50%
- 无 P0 Bug

---

### 3.3 Beta 阶段

**目标**: 功能冻结，进入稳定化

**活动**:
- Bug 修复
- 性能优化
- 文档完善
- B-Gate 检查

**Tag 示例**:
```
v2.8.0-beta1
v2.8.0-beta2
```

**规则**:
- 禁止新功能
- 只允许 Bug Fix
- 所有变更必须走 PR

**入口条件**: B-Gate 通过
**出口条件**:
- 功能冻结完成
- 覆盖率 ≥65%
- 无 P0/P1 Bug

---

### 3.4 RC 阶段

**目标**: 发布候选版本

**活动**:
- 候选版本构建
- 全量测试
- 安全扫描
- R-Gate 检查

**Tag 示例**:
```
v2.8.0-rc1
v2.8.0-rc2
```

**规则**:
- 禁止新功能
- 只允许严重 Bug Fix
- CI 必须全绿

**入口条件**: R-Gate 通过
**出口条件**:
- 测试 100% 通过
- 覆盖率 ≥80%
- 无安全漏洞

---

### 3.5 GA 阶段

**目标**: 正式发布

**活动**:
- 最终版本构建
- 发布审批
- Git Tag 创建
- Release Notes 发布
- 公告发布

**Tag**:
```
v2.8.0
```

**流程**:
```
rc/v2.8.0 ──G-Gate 通过──► main (merge)
                              │
                              ▼
                         Tag v2.8.0
                              │
                              ▼
                    docs/releases/v2.8.0/
                              │
                              ▼
                        公告发布
```

**入口条件**: G-Gate 通过
**出口条件**: 发布审批通过

## 四、发布检查点

### 4.1 Alpha → Beta 检查点

| 检查项 | 要求 | 状态 |
|--------|------|------|
| 功能完整性 | 所有计划功能已实现 | ✅ 完成 |
| 测试覆盖率 | ≥50% | ✅ 完成 (Baseline 226/226) |
| 编译状态 | Release 构建成功 | ✅ 完成 |
| CI 状态 | 所有 Job 通过 | ✅ 完成 |
| 文档状态 | 核心文档完成 | ✅ 完成 |

### 4.2 Beta → RC 检查点

| 检查项 | 要求 | 状态 |
|--------|------|------|
| 功能冻结 | 无新功能开发 | ✅ 已冻结 |
| Bug 状态 | 无 P0/P1 Bug | ✅ 无未修复 P0/P1 |
| 测试覆盖率 | ≥65% | ✅ Baseline 226/226 PASS |
| Clippy | 零警告 | ✅ 通过 |
| 性能测试 | 基本性能达标 | ✅ 通过 |

### 4.3 RC → GA 检查点

| 检查项 | 要求 | 状态 |
|--------|------|------|
| 测试状态 | 100% 通过 | ⏳ |
| 覆盖率 | ≥80% | ⏳ |
| 安全扫描 | 无漏洞 | ⏳ |
| 文档完整 | 所有文档就绪 | ⏳ |
| 发布审批 | 审批通过 | ⏳ |

## 五、Tag 命名规范

### 5.1 格式

```
v{MAJOR}.{MINOR}.{PATCH}-{phase}{number}
```

### 5.2 示例

| 阶段 | Tag 格式 | 示例 |
|------|----------|------|
| Alpha | vX.Y.Z-alpha{N} | v2.8.0-alpha1 |
| Beta | vX.Y.Z-beta{N} | v2.8.0-beta1 |
| RC | vX.Y.Z-rc{N} | v2.8.0-rc1 |
| GA | vX.Y.Z | v2.8.0 |

## 六、版本号规则

MAJOR.MINOR.PATCH
2  . 8  . 0

MAJOR: 破坏性变更

PATCH: Bug 修复 (向后兼容)
```
