# DEVELOPMENT_PLAN_TEMPLATE.md — SQLRustGo 版本开发计划模版

> **版本**: 1.0
> **创建日期**: 2026-05-14
> **用途**: 所有版本 (v2.9.0+) 开发计划必须基于本文档创建
> **SSOT**: 本模版的使用是强制性的，不使用本模版创建的开发计划视为不合格

---

## 一、模版使用说明

### 1.1 强制使用声明

```
❌ 不允许: 自由格式的开发计划
✅ 必须: 基于本文档模版创建
```

每个版本的开发计划必须包含以下章节，且顺序不得改变。

### 1.2 版本开发计划文件命名

```
docs/releases/v{MAJOR}.{MINOR}.{PATCH}/DEVELOPMENT_PLAN.md
```

示例：
- `docs/releases/v3.1.0/DEVELOPMENT_PLAN.md`
- `docs/releases/v3.2.0/DEVELOPMENT_PLAN.md`

### 1.3 必需章节清单

| # | 章节 | 必须包含 |
|---|------|----------|
| 1 | 概述与目标 | 版本目标、范围、关键交付物 |
| 2 | 功能列表 | P0/P1/P2 功能清单，带 Issue 引用 |
| 3 | 技术任务 | 架构变更、依赖升级、工具链 |
| 4 | 测试策略 | 对应测试计划引用 |
| 5 | 门禁计划 | 各阶段门禁入口条件 |
| 6 | 版本延续任务 | 上版本未完成任务映射 |
| 7 | 风险评估 | 已识别风险及缓解措施 |
| 8 | 里程碑 | 关键日期和阶段目标 |
| 9 | 附录 | 详细技术设计文档链接 |

## 三、模版使用检查清单

### 开发计划创建时必须检查

```
□ 文件路径正确: docs/releases/v{VERSION}/DEVELOPMENT_PLAN.md
□ 包含所有 9 个章节
□ 章节顺序正确
□ 所有 TODO 项已填写
□ Issue 引用有效
□ 里程碑日期合理
□ 版本延续任务已映射
□ 门禁入口条件已定义
□ 测试策略与测试计划一致
□ 风险评估已包含
```

### 开发计划审查时必须检查

```
□ 功能列表是否覆盖所有计划的 feature
□ Issue 引用是否有效
□ 里程碑是否与门禁计划一致
□ 版本延续任务是否完整映射上版本未完成任务
□ 风险评估是否覆盖主要风险
□ 测试策略是否与 gate_spec 定义的门禁一致
□ 成功定义是否可验证
□ 文档格式是否符合模版
```
