# 项目目录规范政策

> 定义 SQLRustGo 项目的目录结构、文档存放位置和命名规范

## 1. 目录结构概览

```
┌───────────────────────────────────────────────────────────────────────────┐
│ 根目录 (sqlrustgo/)                                                      │
├───────────────────────────────────────────────────────────────────────────┤
│ ├── docs/                  # 文档根目录                                    │
│ │   ├── governance/        # 治理相关文档                                  │
│ │   ├── plans/             # 规划文档                                      │
│ │   ├── releases/          # 版本发布文档                                   │
│ │   │   └── vX.Y.Z/        # 具体版本的文档                               │
│ │   │       ├── report/    # 版本报告                                     │
│ │   ├── standard/          # 标准和模板                                   │
│ │       └── templates/     # 文档模板                                     │
│ ├── scripts/               # 脚本目录                                      │
│ │   ├── analyze/           # 分析脚本                                    │
│ │   ├── usage/             # 用量分析脚本                                 │
│ │   ├── benchmark/         # 基准测试脚本                                 │
│ │   └── gate/              # 门禁检查脚本                                 │
│ ├── artifacts/             # 构建产物和测试结果                            │
└───────────────────────────────────────────────────────────────────────────┘
```

## 2. 文档存放位置

### 2.1 按文档类型分类

| 文档类型 | 存放位置 | 说明 |
|---------|---------|------|
| 版本发布文档 | `docs/releases/vX.Y.Z/` | 包含特定版本的完整文档 |
| 版本报告 | `docs/releases/vX.Y.Z/report/` | 测试报告、性能分析等 |
| 治理文档 | `docs/governance/` | 权限、流程、规范等治理相关文档 |
| 规划文档 | `docs/plans/` | 项目规划、改进计划等 |
| 标准和模板 | `docs/standard/` | 文档标准和模板文件 |
| 脚本文件 | `scripts/` | 按功能分类存放 |
| 构建产物 | `artifacts/` | CSV、PNG、XLSX等产物 |

### 2.2 版本化文档管理

- **版本目录格式**：`docs/releases/vX.Y.Z`
- **版本号规则**：语义化版本 (MAJOR.MINOR.PATCH)
- **版本归档**：每个版本独立归档，包含完整的文档集合
- **向后兼容**：保留旧版本文档，确保历史链接可访问

## 3. 文档命名规范

### 3.1 通用命名规则

- **使用 PascalCase 或 snake_case**：根据文件类型选择合适的命名方式
- **清晰描述内容**：文件名应准确反映文档内容
- **包含版本信息**：版本相关文档应包含版本号
- **使用英文命名**：便于国际化协作

### 3.2 具体文档类型命名

| 文档类型 | 命名示例 | 说明 |
|---------|---------|------|
| 版本说明 | `README.md` | 版本总览和导航 |
| 变更日志 | `CHANGELOG.md` | 版本变更记录 |
| 测试计划 | `TEST_PLAN.md` | 测试策略和计划 |
| 测试报告 | `TEST_REPORT.md` | 测试执行结果 |
| 性能报告 | `PERFORMANCE_REPORT.md` | 性能测试结果 |
| 安全分析 | `SECURITY_ANALYSIS.md` | 安全评估报告 |
| API 文档 | `API_DOCUMENTATION.md` | API 使用说明 |
| 安装指南 | `INSTALLATION_GUIDE.md` | 安装和配置说明 |
| 开发指南 | `DEVELOPMENT_GUIDE.md` | 开发环境搭建和规范 |
| 升级指南 | `UPGRADE_GUIDE.md` | 版本升级说明 |

## 4. 脚本文件分类

### 4.1 脚本目录结构

| 目录 | 用途 | 示例文件 |
|------|------|---------|
| `scripts/analyze/` | 代码分析脚本 | `analyze_code.sh` |
| `scripts/usage/` | 用量分析脚本 | `analyze_output_usage.py` |
| `scripts/benchmark/` | 基准测试脚本 | `run_benchmark.sh` |
| `scripts/gate/` | 门禁检查脚本 | `check_docs_links.sh` |

### 4.2 脚本命名规范

- **功能描述**：脚本名应清晰描述其功能
- **文件扩展名**：根据脚本类型使用合适的扩展名（.sh, .py 等）
- **版本兼容**：脚本应在主流环境中可执行

## 5. 索引和链接规则

### 5.1 文档索引

- **根目录 README.md**：项目总览和主要文档导航
- **docs/README.md**：文档体系总览和索引
- **版本目录 README.md**：版本特定文档导航

### 5.2 链接规范

- **使用相对路径**：确保链接在不同环境中都能正常工作
- **避免绝对路径**：减少环境依赖
- **定期检查**：使用 `scripts/gate/check_docs_links.sh` 检查链接有效性

## 6. 归档策略

### 6.1 文档生命周期

- **Active**：当前维护的文档版本
- **Archived**：已发布但不再维护的文档版本

### 6.2 归档流程

1. 版本发布后，将文档归档到对应版本目录
2. 更新索引和导航链接
3. 保留历史版本的完整文档

## 7. 执行和验证

### 7.1 门禁检查

- **文档链接检查**：`bash scripts/gate/check_docs_links.sh`
- **格式检查**：`cargo fmt --check --all`
- **代码质量检查**：`cargo clippy --all-features -- -D warnings`

### 7.2 验证流程

1. 提交前执行门禁检查
2. 确保所有文档链接有效
3. 验证文档结构符合规范

## 8. 变更管理

### 8.1 文档更新流程

1. 在对应目录修改或创建文档
2. 更新相关索引和导航
3. 执行门禁检查
4. 提交变更并创建 PR

### 8.2 迁移策略

- **文件迁移**：使用 `git mv` 保持版本历史
- **路径映射**：记录迁移前后的路径映射
- **向后兼容**：保留旧路径的重定向或说明

## 9. 附则

- 本规范适用于所有新增文档和脚本
- 现有文档应逐步迁移到符合本规范的位置
- 特殊情况需经项目维护者批准
