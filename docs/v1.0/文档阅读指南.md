# SQLRustGo v1.0 文档阅读指南

> 本指南帮助你系统地阅读 v1.0 版本文档，理解项目从规划到 Beta 的完整演进过程。

---

## 一、v1.0 版本概览

v1.0 是 SQLRustGo 的首个正式版本，经历了以下阶段：

```
草稿计划 → 草稿实现 → 草稿评估 → Alpha → Beta（当前）
```

每个阶段都有对应的文档，记录了项目演进的真实轨迹。

## 五、常见问题

### Q1: 我应该从哪里开始？

**A**: 如果你是新手，建议按时间顺序阅读，从 [草稿计划/](草稿计划/) 开始。

### Q2: 文档太多，没时间全看怎么办？

**A**: 根据你的角色选择重点文档：
- 学生：阶段性工作报告.md + 综合改进计划.md
- 开发者：AI 协作指南.md + 开发文档.md

### Q3: 想了解某个具体功能是如何实现的？

**A**: 查看 [草稿/阶段性工作报告.md](草稿/阶段性工作报告.md)，里面有功能开发过程的记录。

### Q4: 想参与贡献，应该看什么？

**A**: 
1. [AI 协作指南](草稿计划/2026-02-16-ai-collaboration-guide.md) - 了解协作方式
2. [分支策略](草稿计划/2026-02-16-branch-strategy.md) - 了解分支管理
3. [PR 工作流程](草稿计划/2026-02-16-pr-workflow.md) - 了解提交流程
4. [开发文档](dev/DEVELOP.md) - 了解开发规范
