## 三、开发顺序

### Phase 1: MVCC 骨架 (T-01)

1. 快照隔离实现
2. 版本链管理
3. 可见性判断

### Phase 2: 事务管理器 (T-02)

1. 事务开始/提交/回滚
2. 事务状态追踪
3. 长事务处理

### Phase 3: Planner 优化 (O-01)

1. 子查询展开
2. Join 重排序增强
3. 代价模型调优

### Phase 4: 基准测试 (O-02)

1. TPC-H 基准
2. 回归测试
3. 性能对比
