## 覆盖率难以提高的原因分析

### 1. 架构设计因素
- **物理计划层**: 许多执行器是 trait 实现，测试需要 mock 大量依赖
- **HTTP 服务器**: 异步 HTTP 端点难以在单元测试中覆盖
- **存储引擎**: 文件 I/O 操作难以 mock

### 2. 测试策略因素
- **已实现 120+ 优化器测试**: 覆盖率已从初始的 ~50% 提升至 ~74%
- **集成测试 vs 单元测试**: 部分功能更适合集成测试
- **边界条件**: 许多代码路径是错误处理分支

### 3. 代码结构因素
- **未使用的代码**: #[allow(dead_code)] 标记的代码不需要测试
- **平台特定代码**: 某些条件编译分支无法在测试环境覆盖
