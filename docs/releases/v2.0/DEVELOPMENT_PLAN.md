# SQLRustGo v2.0 开发计划

> **版本**: 2.0
> **日期**: 2026-03-26
> **状态**: 规划中

---

## 1. 版本定位与目标

### 1.1 版本定位

v2.0 定位：**生产级轻量数据库**，可作为 MySQL 的平替，用于中小型业务系统（订单管理，内容管理，物联网数据存储等）。

### 1.2 核心目标

| 目标 | 指标 |
|------|------|
| 性能基准 | 50+ 并发连接、1000+ QPS、P99 延迟 <100ms |
| 高可用 | 主从复制、自动故障转移 |
| 数据可靠性 | 在线备份、校验修复 |
| SQL 兼容性 | 外键、自增、UPSERT、窗口函数 |
| 运维监控 | 慢查询日志、监控指标、动态配置 |
| 代码质量 | ≥80% 代码覆盖率 |

### 1.3 时间线

预计 **6~9 个月**（2026 Q2 ~ 2026 Q4），分四个阶段迭代发布。

---

## 2. 改进领域及优先级

| 优先级 | 领域 | 说明 |
|--------|------|------|
| P0 | 性能与可扩展性 | 必须达标的核心指标 |
| P0 | 高可用与可靠性 | 生产环境不可或缺 |
| P0 | SQL 功能完善 | 补齐 MySQL 常用功能 |
| P1 | 运维与监控 | 提升可维护性 |
| P1 | 安全与权限 | 满足安全合规 |
| P2 | 兼容性与迁移 | 降低使用门槛 |

---

## 3. 详细任务分解

### 3.1 性能与可扩展性（P0）

| 任务 | 描述 | 预期产出 | 工作量 |
|------|------|----------|--------|
| 端到端压力测试 | 使用 sysbench 或自研工具，测试 50/100/200 并发下的 QPS、延迟、资源消耗 | 性能基准报告 | 2 周 |
| 查询缓存优化 | 实现缓存失效策略（基于表修改），增加缓存命中率指标 | 查询缓存模块增强 | 3 周 |
| 索引增强 | 支持组合索引和覆盖索引，优化器根据成本选择索引 | 索引管理器扩展 + 优化器改进 | 4 周 |
| 并行查询 | 对聚合、排序等算子实现并行执行（多线程） | 并行执行框架 | 4 周 |
| 批量写入优化 | 优化 INSERT 批处理，减少锁竞争 | 写入路径性能提升 | 2 周 |
| 内存管理优化 | 实现内存池（arena allocator），减少分配开销 | 内存使用更稳定 | 2 周 |

**里程碑**：达到 1000 QPS (50 并发) 基准，P99 延迟 <100ms。

### 3.2 高可用与可靠性（P0）

| 任务 | 描述 | 预期产出 | 工作量 |
|------|------|----------|--------|
| 主从复制 | 实现异步/半同步复制，支持一主多从 | 复制模块（binlog 格式、IO/SQL 线程） | 6 周 |
| 自动故障转移 | 监控主节点状态，自动提升从库为主 | 高可用管理组件 | 4 周 |
| 物理备份与恢复 | 支持在线热备（基于文件快照），增量备份 | 备份工具 + 恢复工具 | 4 周 |
| 数据校验与修复 | 页面 checksum，定期校验，提供修复命令 | 校验工具 CHECK/REPAIR TABLE | 3 周 |
| WAL 性能优化 | 优化 WAL 写入（组提交、异步刷盘），减少 fsync 开销 | 更低的写入延迟 | 2 周 |

**里程碑**：完成主从复制原型，通过故障切换测试，支持热备。

### 3.3 SQL 功能完善（P0）

| 任务 | 描述 | 预期产出 | 工作量 |
|------|------|----------|--------|
| 外键约束 | 实现 FOREIGN KEY 完整语义（级联删除/更新） | 外键约束支持 | 3 周 |
| 自增列 | 实现 AUTO_INCREMENT 及 LAST_INSERT_ID() | 自增列功能 | 2 周 |
| UPSERT | 支持 INSERT ... ON DUPLICATE KEY UPDATE | UPSERT 语法与实现 | 2 周 |
| 完整 JOIN 支持 | 完成 RIGHT JOIN / CROSS JOIN | 物理计划增强 | 2 周 |
| 窗口函数 | 实现 ROW_NUMBER(), RANK(), SUM/AVG OVER | 窗口函数框架 | 5 周 |
| 存储过程/函数（可选） | 支持基础存储过程（无事务） | 存储过程基础能力 | 6 周 |

**里程碑**：通过 MySQL 兼容性测试套件的基础子集。

### 3.4 运维与监控（P1）

| 任务 | 描述 | 预期产出 | 工作量 |
|------|------|----------|--------|
| 慢查询日志 | 记录超过阈值的 SQL，支持分析工具 | 慢查询日志模块 | 2 周 |
| 错误日志 | 分类分级，支持日志轮转 | 错误日志完善 | 1 周 |
| 监控指标 | 暴露 Prometheus 指标（QPS、连接数、缓存命中率、锁等待） | 指标端点 + 集成文档 | 3 周 |
| 动态配置 | 支持运行时修改参数（如 buffer pool 大小） | 配置热更新能力 | 2 周 |
| 诊断命令 | 实现 SHOW STATUS, SHOW PROCESSLIST, KILL | 诊断接口 | 2 周 |

**里程碑**：可通过外部监控系统观察数据库状态，支持动态调整关键参数。

### 3.5 安全与权限（P1）

| 任务 | 描述 | 预期产出 | 工作量 |
|------|------|----------|--------|
| 用户与角色管理 | 支持 CREATE USER, GRANT, REVOKE | 权限管理模块 | 4 周 |
| SSL/TLS 支持 | 加密客户端连接 | SSL 配置选项 | 2 周 |
| 密码加密存储 | 使用 SCRAM 或 bcrypt 存储密码 | 认证机制升级 | 1 周 |
| 审计日志 | 记录 DDL/DML 操作，支持筛选 | 审计日志模块 | 3 周 |

**里程碑**：通过基本安全测试，支持用户隔离。

### 3.6 兼容性与迁移（P2）

| 任务 | 描述 | 预期产出 | 工作量 |
|------|------|----------|--------|
| MySQL 语法兼容模式 | 支持 REPLACE, INSERT IGNORE, SHOW 等 | 兼容性增强 | 3 周 |
| 数据迁移工具 | 提供 mysqldump 导入工具，支持在线迁移 | 迁移脚本 + 文档 | 2 周 |
| 驱动增强 | 完善 JDBC/ODBC 驱动，通过常见 ORM 测试 | 驱动兼容性报告 | 2 周 |

**里程碑**：可以无缝迁移小型 MySQL 应用。

### 3.7 测试与质量（贯穿始终）

| 任务 | 描述 | 预期产出 | 工作量 |
|------|------|----------|--------|
| 代码覆盖率提升至 80% | 补充单元测试，重点覆盖事务、存储、优化器 | 覆盖率报告 | 持续 |
| 混沌测试 | 注入故障（磁盘满、进程崩溃、网络分区） | 混沌测试框架 + 测试报告 | 4 周 |
| 长期稳定性测试 | 7×24 小时混合负载运行 | 稳定性报告 | 持续 |
| SQL 模糊测试 | 使用 SQLancer 或 SQLsmith 发现潜在错误 | 发现的 Bug 修复 | 持续 |

**里程碑**：所有测试通过率 100%，无已知严重 Bug。

## 5. 风险评估与应对

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 复制实现复杂，延期 | 中 | 高 | 先实现异步复制，简化日志格式 |
| 窗口函数性能不达标 | 中 | 中 | 采用离线计算或限制使用场景 |
| 代码覆盖率目标难以达成 | 低 | 中 | 引入自动化测试生成工具 |
| 团队资源不足 | 中 | 高 | 明确功能优先级，按需裁剪 P2 任务 |
| 兼容性测试发现大量差异 | 中 | 中 | 建立 MySQL 测试套件，逐步修复 |
