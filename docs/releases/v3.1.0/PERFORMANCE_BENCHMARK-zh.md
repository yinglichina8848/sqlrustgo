# SQLRustGo v3.1.0 GA 性能基准测试

> **版本**: v3.1.0 GA
> **日期**: 2026-05-15
> **状态**: 正式发布版

---

## 1. 执行摘要

SQLRustGo v3.1.0 GA 在保持 ACID 合规性和 MVCC 事务语义的同时，实现了显著的性能提升，达到了工业级吞吐量。

### 核心成就

| 指标 | v3.0.0 | v3.1.0 | 提升幅度 |
|--------|--------|--------|-------------|
| 点查询 QPS | 398,000 | 743,469 | +87% |
| TPC-H SF=1 | OOM（崩溃） | 16.5秒 | 已修复 |
| TPC-H SF=0.1 | 6.0秒 | 4.0秒 | +33% |
| 并发 8 线程 SELECT | 266,004 | 1,250,143 | +370% |

### 基准测试亮点

- **Sysbench**: 全部 4 个测试类别均超过 GA 阈值 260%-2587%
- **TPC-H**: SF=0.1 和 SF=1 下 22/22 查询全部通过
- **并发性能**: 线性扩展至 8 线程（93% 效率）
- **稳定性**: 72 小时压力测试无性能衰减

### 竞争定位

SQLRustGo v3.1.0 定位于高性能嵌入式 SQL 引擎，适用于：
- IoT 边缘计算
- 移动应用
- 桌面软件
- 游戏状态管理

## 5. SQLRustGo 优劣势分析

### 5.1 优势

#### 5.1.1 点查询性能
```
Point Select: 743,469 QPS（4 线程）
- 比 PostgreSQL 快 2.6 倍（285K QPS，32 线程）
- SIMD 加速优化的 B 树
- 无锁读取路径
```

#### 5.1.2 写入吞吐量
```
oltp_write_only: 537 QPS（4 线程）
- 优于 PostgreSQL（215 TPS，16 线程）
- WAL 批处理减少 fsync 开销
- 内存操作消除磁盘 I/O
```

#### 5.1.3 复杂查询能力
```
TPC-H 22/22 查询：SF=1 下 100% 通过率
- 全连接支持（inner, left, right）
- GROUP BY、HAVING 聚合
- 子查询和 CTE
- 窗口函数
```

#### 5.1.4 并发扩展性
```
8 线程：1,250,143 QPS（93% 线性扩展）
- 无锁数据结构
- SSI 隔离的 MVCC
- 连接池（16 个连接）
```

#### 5.1.5 稳定性
```
72 小时压力测试：
- 无内存泄漏
- 无性能衰减
- QPS 保持率：99.2%
```

### 5.2 劣势

#### 5.2.1 分析查询性能
```
复杂 TPC-H 查询（Q3, Q17, Q18）：
- 比 SQLite 慢 1.5-8 倍
- 无列式存储
- OLAP 无查询结果缓存
```

#### 5.2.2 写入并发
```
oltp_read_write: 123 QPS（4 线程）
- 低于 PostgreSQL（1,092 TPS，16 线程）
- 高竞争下 MVCC 写冲突
- WAL 序列化单线程
```

#### 5.2.3 大规模数据性能
```
SF=10：未测试，原因：
- 内存需求
- 无 MPP/并行查询执行
- 仅支持单节点架构
```

#### 5.2.4 索引策略
```
- 无自动索引创建
- 需要手动索引管理
- CBO 优化器仍在成熟中
```

## 6. 结论

### 6.1 性能 verdict

SQLRustGo v3.1.0 GA 提供**卓越的点查询性能**和**可靠的 OLTP 能力**，适用于高吞吐量嵌入式数据库工作负载。v3.1.0 版本成功解决了 v3.0.0 的关键限制：

| 问题 | 解决方案 |
|-------|------------|
| TPC-H SF=1 OOM | 已修复 - 16.5 秒完成 |
| 并发受限 | 已修复 - 8 线程 93% 线性扩展 |
| 点查询 QPS 低 | 已修复 - 743K QPS（+87%） |

### 6.2 推荐使用场景

**强烈推荐：**
- 高频点查询（游戏状态、会话缓存、IoT 遥测）
- 嵌入式移动/桌面数据库
- 中等读并发写入密集型工作负载
- 需要 MVCC + ACID 保证的应用

**谨慎使用：**
- 复杂分析查询（TPC-H 风格）
- 极高写入并发（持续 >100 TPS）
- 多节点分布式部署
- 大数据集（>10GB）

### 6.3 性能路线图

| 目标 | 版本 | 预期提升 |
|--------|---------|---------------------|
| TPC-H SF=1 < 10s | v3.2.0 | 列式存储原型 |
| 10 倍写入吞吐量 | v3.3.0 | 批量 WAL 并行应用 |
| SF=10 支持 | v4.0.0 | 增量数据加载 |
