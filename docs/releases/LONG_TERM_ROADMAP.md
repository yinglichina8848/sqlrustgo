# SQLRustGo 版本演化计划 v3.0

> **版本**: v3.0
> **更新日期**: 2026-03-13
> **制定依据**: 综合前期讨论及最新工程评审意见
> **核心原则**: 版本语义化、工程现实性、风险可控、可观测性内建

---

## 一、版本号语义与成熟度目标

| 主版本 | 技术范式 | 成熟度等级 | 核心特征 | 对标系统 |
| --- | --------- | ------------------- | -------------------------- | ------------------ |
| 1.x | 单机基础数据库 | L3 (Mini DBMS) | 火山模型、行存、B+Tree、MVCC、SQL-92 | early PostgreSQL |
| 2.x | 单机高性能分析引擎 | L4 (Analytical DB) | 向量化执行、列存、CBO、并行执行 | DuckDB |
| 3.x | 分布式数据库 | L5 (Distributed DB) | 分布式执行、分片、共识、分布式事务 | TiDB / CockroachDB |

**演进路径**: 1.x（可用）→ 2.x（快）→ 3.x（大）

---

## 二、版本演化总览（2026–2029+）

### 🔹 v1.x 系列：单机基础数据库（2026–2028）

| 版本 | 代号 | 时间 | 核心目标 | 主要功能改进 | 可观测性增强 |
| --------- | -------------- | ------- | ------------------------------- | ----------------------------------------------------------------------------------------- | ----------------------------- |
| **v1.3** | Executor 稳定化 | 2026-05 | 火山模型执行器打牢基础 | Volcano trait、TableScan/Projection/Filter/HashJoin、测试覆盖率（整体≥65%） | 基础日志、Metrics trait、/health 端点 |
| **v1.4** | SQL 基础 + 表达式框架 | 2026-09 | 完整 SQL 查询 + 表达式系统 + LogicalPlan | GROUP BY、聚合、ORDER BY、LIMIT；表达式框架（BinaryExpr, ColumnRef, Constant）；引入LogicalPlan；谓词下推、投影裁剪 | EXPLAIN ANALYZE、慢查询日志（>1s） |
| **v1.5** | 存储引擎 + WAL | 2027-01 | 持久化存储与预写日志 | 页式存储、缓冲池、WAL基础；表堆；基础统计信息（表行数） | 存储指标（页读写、缓冲池命中率） |
| **v1.6** | MVCC + 统计信息 | 2027-05 | MVCC 骨架 + 统计信息收集 | MVCC 快照隔离、事务管理器；ANALYZE 命令、列NDV，空值比例，直方图 | 事务指标（活跃事务、提交/中止） |
| **v1.7** | 索引支持 | 2027-08 | B+Tree 索引，提升点查性能 | B+Tree 索引结构、IndexScan 算子、索引维护 | 索引访问指标 |
| **v1.8** | Planner 优化 | 2027-11 | 物理计划生成与规则优化增强 | PhysicalPlan 分离、表达式求值完善、常量折叠、子查询展开 | 优化器基础指标（规则触发次数） |
| **v1.9** | 并发控制 | 2028-02 | 完整 MVCC 隔离级别 + 死锁检测 | 读已提交、可重复读、锁管理器、死锁检测、事务日志恢复（ARIES） | 锁等待直方图、死锁计数器 |
| **v1.10** | 稳定与基准测试 | 2028-05 | 系统稳定性加固 + 性能基准 | Bug修复、性能调优、基准测试套件、回归检测 | 基准报告、稳定性指标 |

### 🚀 v2.x 系列：单机高性能分析引擎（2028–2029）

| 版本 | 代号 | 时间 | 核心目标 | 主要功能改进 | 可观测性增强 |
| -------- | --------- | ------- | --------------------- | --------------------------------------------------- | -------------------------------- |
| **v2.0** | 向量化执行框架 | 2028-08 | 从火山模型升级为向量化执行 | DataChunk 向量化格式、算子向量化版本（Scan/Filter/Join/Aggregate） | 向量化算子性能指标 |
| **v2.1** | 列式存储引擎 | 2028-11 | 增加列存支持，优化分析 I/O | 列存表、Parquet 导入导出、列式压缩、投影下推 | 列存 I/O 指标、压缩率监控 |
| **v2.2** | CBO 成本优化器 | 2029-02 | 引入 Cascades 框架，基于代价优化 | Memo 结构、Join 重排、代价模型（利用统计信息）、规则系统增强 | 优化器追踪（rule fired, cost estimate） |
| **v2.3** | 单机并行执行 | 2029-05 | 多核并行加速查询 | 并行扫描、并行 HashJoin、调度器，工作线程池 | 并行指标（worker 数、shuffle 数据量） |

### 🌐 v3.x 系列：分布式数据库（远期探索，2029+）

| 版本 | 代号 | 时间 | 核心目标 | 主要功能改进 | 可观测性增强 |
| -------- | -------- | ------- | ------------ | ---------------------------------------- | ----------------------------- |
| **v3.0** | 分布式执行框架 | 2029-09 | 支持多节点并行查询 | Exchange 算子、远程分片查询、基础数据分发、节点通信协议 | 分布式追踪（OpenTelemetry + Jaeger） |
| **v3.1** | 分布式事务与容错 | 2030-01 | 跨节点事务一致性与高可用 | 分布式事务（2PC/Percolator）、Raft 共识、自动故障转移、多副本 | 分布式事务指标、共识延迟 |
| **v3.2** | 分布式优化器 | 2030-05 | 生成高效的分布式执行计划 | 跨节点代价估算、数据分布感知、复杂查询支持 | 优化器分布式决策追踪 |

**重要说明**: v3.x 作为长期研究探索方向，实际开发将基于项目资源和社区发展动态推进，不承诺固定发布日期。

## 三、关键改进点说明（针对评审意见）

| 评审意见 | 采纳调整 |
| -------------------- | -------------------------------------------------------- |
| Planner 重构太晚 | v1.4 提前引入 LogicalPlan，v1.8 再进一步优化物理计划与规则 |
| 统计信息太晚 | v1.5 增加基础统计（表行数），v1.6 完成完整统计信息 |
| 缺少 Expression Engine | v1.4 明确加入表达式框架（BinaryExpr, ColumnRef, Constant 等），作为独立模块 |
| Transaction 时间低估 | v1.6 只做 MVCC 骨架，v1.9 完成完整并发控制，周期适当拉长（4个月/版本） |
| 增加 v1.10 稳定版本 | 新增 v1.10，聚焦稳定性、基准测试、Bug 修复，为 v2.0 铺路 |
| 慢查询日志 | v1.4 的 Query Profiler 中包含慢查询阈值记录 |
| 分布式作为远期探索 | v3.x 标记为长期研究，不强制时间线 |
