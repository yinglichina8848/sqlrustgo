## 四、阶段门禁规则

详见 [BRANCH_STAGE_GOVERNANCE.md](./v1.2.0/BRANCH_STAGE_GOVERNANCE.md)

| 阶段 | 门禁要求 | 允许提交类型 |
|------|----------|--------------|
| **Draft** | 编译通过 | 架构、目录、接口设计 |
| **Alpha** | 测试通过率 ≥ 80% | 新功能、新模块 |
| **Beta** | 测试通过率 ≥ 95%、Clippy 零警告 | Bug 修复、性能优化 |
| **RC** | 测试 100% 通过、CI 全绿 | 仅 Critical Bug 修复 |
| **GA** | 所有门禁通过 | 禁止修改 |

## 五、分支管理规范

详见 [BRANCH_STAGE_GOVERNANCE.md](./v1.2.0/BRANCH_STAGE_GOVERNANCE.md)

### 5.1 分支命名规范

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 开发分支 | `develop-v1.x.0` | `develop/v1.3.0` |
| 功能分支 | `feature/v1.x.0-<功能名>` | `feature/v1.3.0-cbo-optimizer` |
| 修复分支 | `fix/v1.x.0-<问题描述>` | `fix/v1.3.0-metrics-bug` |
| 文档分支 | `docs/v1.x.0-<文档类型>` | `docs/v1.3.0-release-notes` |
| 维护分支 | `release/vx.y.z` | `release/v1.3.0` |

### 5.2 分支保护规则

| 分支 | 保护规则 |
|------|----------|
| `main` | 禁止直接 push，必须通过 PR + 2人审核 |
| `develop/vx.y.z` | 禁止直接 push，必须通过 PR + 1人审核 |
| `feature/*`, `fix/*` | 允许直接 push，但需要 CI 通过 |
