# SQLRustGo v2.7.0 向 MySQL 5.7 水平增强开发计划

> **版本**: v2.7.0
> **目标**: 达到 MySQL 5.7 生产就绪水平
> **创建日期**: 2026-04-22

---

## 一、目标概述

### 1.1 背景

根据 v2.6.0 综合评估，v2.6.0 版本已达到 MySQL 5.7 约 75-80% 能力水平，已完成 MVCC SSI、存储过程集成、触发器集成等核心功能。v2.7.0 目标是在此基础上继续完善，达到 85-90% 能力水平。

已完成 (v2.6.0):
- MVCC SSI 事务隔离级别实现
- 存储过程集成
- 触发器完整集成
- CBO 优化器启用
- FULL OUTER JOIN 支持

### 1.2 目标版本

- **版本**: v2.7.0 (目标 GA)
- **目标**: 达到 MySQL 5.7 85%+ 能力
- **发布时间**: 2026-06-30

| 债务项 | 影响 | 优先级 |

## 八、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 主从复制实现复杂 | 进度延迟 | 预留缓冲时间，优先保证数据一致性 |
| 分区表裁剪优化难度大 | 进度延迟 | 分阶段实现，先基础后优化 |
| 故障转移测试覆盖不足 | 生产故障 | 延长测试周期，增加故障注入测试 |
| 物化视图刷新性能 | 查询结果不一致 | 提供多种刷新策略 |
