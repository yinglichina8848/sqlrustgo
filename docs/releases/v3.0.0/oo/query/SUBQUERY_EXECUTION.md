# 子查询执行链路

> Subquery: 标量子查询、EXISTS、IN、CTE

## 1. 子查询概述

### 1.1 子查询类型

```
┌─────────────────────────────────────────────────────────────┐
│                    子查询分类                                 │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────┐ │
│  │   标量子查询    │  │   EXISTS       │  │    IN       │ │
│  │                 │  │                 │  │             │ │
│  │ SELECT MAX(x)  │  │ WHERE EXISTS   │  │ WHERE x IN │ │
│  │ FROM t WHERE   │  │ (SELECT ...)   │  │ (SELECT ...)│ │
│  │   y = (SELECT) │  │                 │  │             │ │
│  └─────────────────┘  └─────────────────┘  └─────────────┘ │
│                                                              │
│  ┌─────────────────┐  ┌─────────────────┐                   │
│  │   表子查询      │  │   CTE          │                   │
│  │                 │  │                 │                   │
│  │ FROM (SELECT)  │  │ WITH cte AS   │                   │
│  │   AS t         │  │ (SELECT ...)   │                   │
│  └─────────────────┘  └─────────────────┘                   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 子查询表达式结构

```rust
// crates/expr/src/expr.rs
pub enum Subquery {
    /// 单行子查询 (标量上下文)
    Scalar,
    /// EXISTS 检查
    Exists,
    /// IN 列表检查
    In,
}

pub struct SubqueryExpr {
    pub query: Subquery,
}
```

## 3. 子查询执行策略

### 3.1 标量子查询执行

```
┌─────────────────────────────────────────────────────────────┐
│                 标量子查询执行                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  SELECT * FROM orders                                      │
│  WHERE price > (SELECT AVG(price) FROM products)           │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              策略 1: 标量物化                       │   │
│  │  ─────────────────────────────────────────────────  │   │
│  │                                                      │   │
│  │  1. 执行子查询: SELECT AVG(price) FROM products    │   │
│  │  2. 物化结果: avg_price = 25.5                     │   │
│  │  3. 执行主查询: SELECT * FROM orders               │   │
│  │     WHERE price > 25.5                             │   │
│  │                                                      │   │
│  │  优点: 简单易实现                                   │   │
│  │  缺点: 子查询结果变化时需重新计算                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              策略 2: 标量相关                        │   │
│  │  ─────────────────────────────────────────────────  │   │
│  │                                                      │   │
│  │  SELECT * FROM orders o                            │   │
│  │  WHERE price > (                                   │   │
│  │    SELECT AVG(price) FROM products p               │   │
│  │    WHERE p.category = o.category                    │   │
│  │  )                                                  │   │
│  │                                                      │   │
│  │  每一行 orders 都需要重新执行子查询                   │   │
│  │  (相关子查询)                                       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 EXISTS 执行

```
┌─────────────────────────────────────────────────────────────┐
│                    EXISTS 执行                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  SELECT * FROM customers c                                │
│  WHERE EXISTS (                                           │
│    SELECT 1 FROM orders o WHERE o.customer_id = c.id    │
│  )                                                        │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              EXISTS 语义                             │   │
│  │  ─────────────────────────────────────────────────  │   │
│  │                                                      │   │
│  │  EXISTS(subquery) = TRUE  if 子查询返回至少一行     │   │
│  │  EXISTS(subquery) = FALSE if 子查询返回零行        │   │
│  │                                                      │   │
│  │  优化: 找到第一行即可终止                          │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  执行流程:                                                 │
│  ─────────────────────────────────────────────────────────  │
│  for each customer c:                                     │
│      execute subquery with c.id                           │
│      if subquery returns row:                            │
│          emit c                                           │
│      else                                                 │
│          skip c                                          │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 IN 执行

```
┌─────────────────────────────────────────────────────────────┐
│                    IN 执行                                   │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  SELECT * FROM orders                                     │
│  WHERE customer_id IN (                                   │
│    SELECT id FROM customers WHERE city = 'Beijing'        │
│  )                                                        │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              IN 语义                                  │   │
│  │  ─────────────────────────────────────────────────  │   │
│  │                                                      │   │
│  │  value IN (set) = TRUE  if value ∈ set             │   │
│  │  value IN (set) = FALSE if value ∉ set            │   │
│  │                                                      │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  策略 1: 物化 IN 列表                                     │
│  ─────────────────────────────────────────────────────────  │
│  1. 执行子查询，获取所有 customer_id: {1, 5, 8, 12}     │
│  2. 构建 Hash Set: {1, 5, 8, 12}                        │
│  3. 扫描 orders，对每行检查 customer_id 是否在 Set 中      │
│                                                              │
│  策略 2: 半连接 (Semi Join)                               │
│  ─────────────────────────────────────────────────────────  │
│  SELECT * FROM orders o                                   │
│  WHERE EXISTS (                                          │
│    SELECT 1 FROM customers c                             │
│    WHERE c.city = 'Beijing' AND c.id = o.customer_id    │
│  )                                                        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 5. CTE (Common Table Expression)

### 5.1 CTE 架构

```
┌─────────────────────────────────────────────────────────────┐
│                    CTE 架构                                 │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  WITH                                                       │
│    cte1 AS (                                               │
│      SELECT * FROM t1 WHERE x > 10                        │
│    ),                                                       │
│    cte2 AS (                                               │
│      SELECT * FROM t2 WHERE y < 20                         │
│    )                                                        │
│  SELECT * FROM cte1 JOIN cte2 ON cte1.id = cte2.id       │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              CTE 处理流程                              │   │
│  │  ─────────────────────────────────────────────────── │   │
│  │                                                      │   │
│  │  1. Parser: 解析 CTE 定义，建立 CTE 作用域           │   │
│  │  2. Planner: 展开 CTE 为子查询                       │   │
│  │  3. Optimizer: 决定物化还是内联                      │   │
│  │  4. Executor: 执行查询                               │   │
│  │                                                      │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 递归 CTE

```sql
-- 递归 CTE 示例: 计算组织架构
WITH RECURSIVE org_chain AS (
    -- 基础用例: CEO
    SELECT id, name, manager_id, 1 as level
    FROM employees
    WHERE manager_id IS NULL

    UNION ALL

    -- 递归用例: 员工
    SELECT e.id, e.name, e.manager_id, oc.level + 1
    FROM employees e
    JOIN org_chain oc ON e.manager_id = oc.id
)
SELECT * FROM org_chain;
```

### 5.3 CTE 递归执行状态机

```
                  ┌──────────────────┐
                  │    INITIAL       │
                  └────────┬─────────┘
                           │ parse WITH clause
                           ▼
                  ┌──────────────────┐
                  │   ANCHOR_MEM   │
                  └────────┬─────────┘
                           │ execute anchor member
                           ▼
                  ┌──────────────────┐
                  │   RECURSIVE     │
                  └────────┬─────────┘
                           │ join with previous result
                           ▼
                  ┌──────────────────┐
                  │   UNION_ALL     │
                  └────────┬─────────┘
                           │ append new rows
                           ▼
                  ┌──────────────────┐
                  │   NO_NEW_ROWS   │
                  └────────┬─────────┘
                           │ no more rows
                           ▼
                  ┌──────────────────┐
                  │     DONE        │
                  └──────────────────┘
```

## 7. 覆盖率差距分析

### 7.1 当前覆盖率

| 组件 | 行覆盖率 | 说明 |
|------|----------|------|
| expr.rs (SubqueryExpr) | ~60% | 基础结构 |
| planner.rs (subquery planning) | ~55% | 规划逻辑 |
| eval.rs (subquery eval) | ~50% | 执行逻辑 |
| CTE support | ~55% | WITH 展开 |

### 7.2 差距原因

1. **相关子查询优化不足**: 每行都重新执行子查询
2. **CTE 物化策略简单**: 无智能选择物化还是内联
3. **递归 CTE 支持有限**: 深度限制未实现
4. **LATERAL 子查询**: 未实现

### 7.3 提升计划

| 阶段 | 任务 | 目标覆盖率 |
|------|------|-----------|
| v3.1.0 | 相关子查询改写为 JOIN | 70% |
| v3.1.0 | CTE 物化优化 | 75% |
| v3.2.0 | LATERAL 子查询 | 60% |
