# DDL 执行链路

> Data Definition Language: CREATE, DROP, ALTER, TRUNCATE

## 1. DDL 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                        DDL 执行架构                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  CREATE TABLE ──┬──► Parser ──► Catalog.check()            │
│                  │        │                                │
│  DROP TABLE ────┼──► Executor ──► Storage.create_table()    │
│                  │        │                                │
│  ALTER TABLE ───┘        │                                │
│                           ▼                                │
│                    Metadata Registry                        │
│                    (数据字典)                               │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 2. CREATE TABLE 执行链路

### 2.1 CREATE TABLE 时序图

```
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    age INT DEFAULT 18,
    email VARCHAR(255) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
    │
    ▼
┌─────────────────────────────────────────────┐
│              Parser                          │
│  CREATE TABLE Statement {                     │
│    table_name: "users",                     │
│    columns: [                               │
│      {name: "id", type: INT, pk: true},     │
│      {name: "name", type: VARCHAR, nn: true},│
│      {name: "age", type: INT, default: 18}, │
│      {name: "email", type: VARCHAR, unique}, │
│      {name: "created_at", type: TIMESTAMP}  │
│    ]                                        │
│  }                                          │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Catalog                        │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 检查表名是否已存在                 │  │
│  │ 2. 检查列名是否重复                 │  │
│  │ 3. 检查约束合法性                   │  │
│  │ 4. 注册表元数据                      │  │
│  │ 5. 分配表 ID                        │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Storage                        │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 创建表数据文件 (*.tbl)           │  │
│  │ 2. 创建主键索引文件                  │  │
│  │ 3. 创建唯一索引文件                  │  │
│  │ 4. 写入系统表                        │  │
│  │ 5. 写 DDL WAL 日志                  │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

## 3. DROP TABLE 执行链路

### 3.1 DROP TABLE 时序图

```
DROP TABLE users
    │
    ▼
┌─────────────────────────────────────────────┐
│              Catalog                        │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 检查表是否存在                    │  │
│  │ 2. 检查是否有依赖视图               │  │
│  │ 3. 检查是否有外键依赖               │  │
│  │ 4. 获取表 ID                        │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Storage                        │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 删除表数据文件                    │  │
│  │ 2. 删除所有索引文件                  │  │
│  │ 3. 从系统表移除元数据                │  │
│  │ 4. 级联删除依赖视图                  │  │
│  │ 5. 级联删除外键约束                  │  │
│  │ 6. 写 DDL WAL 日志                  │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

### 3.2 DROP TABLE 状态图

```
        ┌──────────┐
        │  CHECK   │
        │ EXIST   │
        └────┬─────┘
             │
    ┌────────▼────────┐
    │  CHECK FOREIGN │──── Constraint Found ──► ┌──────────────┐
    │    KEYS       │                         │CASCADE_DROP │
    └────────┬────────┘                         └──────────────┘
             │ No Constraints
    ┌────────▼────────┐
    │  ACQUIRE LOCK  │
    └────────┬────────┘
             │
    ┌────────▼────────┐
    │ DELETE_FILES   │
    │ (data + index)│
    └────────┬────────┘
             │
    ┌────────▼────────┐
    │ REMOVE_METADATA │
    │   FROM CATALOG  │
    └────────┬────────┘
             │
    ┌────────▼────────┐
    │   WRITE_DDL_WAL │
    └────────┬────────┘
             │
    ┌────────▼────────┐
    │     COMMIT      │
    └────────────────┘
```

## 4. ALTER TABLE 执行链路

### 4.1 ALTER TABLE 操作类型

| 操作 | 说明 | 复杂度 |
|------|------|----------|
| ADD COLUMN | 添加新列 | O(1) 元数据 |
| DROP COLUMN | 删除列 | O(n) 数据重构 |
| MODIFY COLUMN | 修改列类型 | O(n) 数据转换 |
| ADD INDEX | 添加索引 | O(n) 索引构建 |
| ADD CONSTRAINT | 添加约束 | O(1) 元数据 |

### 4.2 ALTER TABLE 时序图

```
ALTER TABLE users ADD COLUMN phone VARCHAR(20)
    │
    ▼
┌─────────────────────────────────────────────┐
│              Parser                          │
│  ALTER TABLE {                              │
│    operation: ADD_COLUMN,                   │
│    column: {name: "phone", type: VARCHAR}   │
│  }                                          │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Catalog                        │
│  - 检查列名不存在                          │
│  - 更新表 schema                          │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Storage                        │
│  ┌─────────────────────────────────────┐  │
│  │ ADD COLUMN:                         │  │
│  │   - 更新系统表                       │  │
│  │   - 不需要移动数据                    │  │
│  │                                      │  │
│  │ DROP/MODIFY COLUMN:                  │  │
│  │   - 遍历所有数据页                   │  │
│  │   - 转换每行数据                      │  │
│  │   - 更新所有索引                      │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

## 5. DDL 与事务

### 5.1 DDL 事务特性

```
┌─────────────────────────────────────────────────────────────┐
│                    DDL 事务处理                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  DDL 语句自动提交 (Auto-commit)                              │
│  ─────────────────────────────────                         │
│  - 每个 DDL 语句是一个独立事务                               │
│  - DDL 执行前隐式 COMMIT                                  │
│  - DDL 执行后自动 COMMIT                                  │
│                                                              │
│  DDL WAL 特殊处理                                          │
│  ─────────────────────────────────                         │
│  - DDL 日志记录完整的 DDL 语句                             │
│  - 恢复时重做 (REDO) DDL                                  │
│  - 不支持 UNDO DDL (通过反操作补偿)                       │
│                                                              │
│  CREATE ──► COMMIT ──► DDL WAL                          │
│  DROP ─────► COMMIT ──► DDL WAL ──► 反向补偿              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 7. 覆盖率差距分析

| 操作 | 当前覆盖率 | 目标 | 差距 |
|------|------------|------|------|
| CREATE TABLE | 60% | 90% | 30% |
| DROP TABLE | 55% | 85% | 30% |
| ALTER ADD | 50% | 85% | 35% |
| ALTER DROP | 40% | 80% | 40% |
| CREATE INDEX | 65% | 90% | 25% |
| DROP INDEX | 60% | 85% | 25% |

## 8. 性能问题与优化

### 8.1 已知性能问题

| 问题 | 影响 | 优先级 |
|------|------|----------|
| ALTER TABLE 重构全表 | 大表 DDL 极慢 | P0 |
| DROP TABLE 级联检查慢 | 外键多时删除慢 | P1 |
| DDL 无并发支持 | DDL 阻塞 DML | P2 |

### 8.2 优化建议

```
┌─────────────────────────────────────────────────────────────┐
│                    DDL 性能优化建议                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 快速 DDL (Instant DDL)                                 │
│     - ADD COLUMN 只修改元数据，不移动数据                     │
│     - MySQL 5.6+ 支持                                       │
│                                                              │
│  2. 并行 DDL                                                │
│     - ALTER TABLE ADD INDEX 并行构建                        │
│     - 利用多核 CPU                                           │
│                                                              │
│  3. DDL 事务优化                                           │
│     - DDL 不阻塞 DML                                        │
│     - 元数据锁细化                                           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```
