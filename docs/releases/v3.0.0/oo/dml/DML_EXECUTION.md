# DML 执行链路

> Data Manipulation Language: INSERT, UPDATE, DELETE, SELECT

## 1. DML 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                        DML 执行架构                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  INSERT ──┬──► Parser ──► Planner ──► Optimizer ──►        │
│           │                                                    │
│  UPDATE ──┼──► Executor ──► Storage ──► WAL                │
│           │                         │                        │
│  DELETE ──┘                         ▼                        │
│                               Transaction                    │
│                                                          │
│  SELECT ───────────────────────────┘                        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 2. INSERT 执行链路

### 2.1 INSERT 时序图

```
INSERT INTO users (id, name, age) VALUES (1, 'Alice', 30)
    │
    ▼
┌─────────────────────────────────────────────┐
│              Parser                           │
│  INSERT {                                    │
│    table: "users",                          │
│    columns: [id, name, age],                 │
│    values: [1, 'Alice', 30]                 │
│  }                                          │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Planner                        │
│  - 解析表结构                              │
│  - 绑定列类型                              │
│  - 生成 InsertNode                         │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Executor                        │
│  InsertExecutor::execute()                  │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 验证约束 (主键、唯一、外键)        │  │
│  │ 2. 构建 Row                            │  │
│  │ 3. 调用 Storage.insert_row()           │  │
│  │ 4. 更新索引                            │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Storage                         │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 获取表文件锁                       │  │
│  │ 2. 定位插入位置                        │  │
│  │ 3. 写入数据页                          │  │
│  │ 4. 更新 B+Tree 索引                   │  │
│  │ 5. 写 WAL 日志                        │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

## 3. UPDATE 执行链路

### 3.1 UPDATE 时序图

```
UPDATE users SET age = 31 WHERE id = 1
    │
    ▼
┌─────────────────────────────────────────────┐
│              Parser                          │
│  UPDATE {                                   │
│    table: "users",                         │
│    set: {age = 31},                        │
│    where: id = 1                             │
│  }                                          │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Executor                        │
│  UpdateExecutor::execute()                  │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 定位要更新的行 (通过索引或扫描)    │  │
│  │ 2. 读取旧行并记录到事务               │  │
│  │ 3. 写入新行                           │  │
│  │ 4. 更新所有受影响索引                │  │
│  │ 5. 写 WAL 日志                        │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

### 3.2 UPDATE 两种策略

```
┌─────────────────────────────────────────────────────────┐
│                   UPDATE 策略选择                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  策略 A: 原地更新 (In-place Update)                     │
│  ─────────────────────────────────────────              │
│  适用条件:                                               │
│  - 更新的列不包含索引键                                   │
│  - 行大小不变                                            │
│  流程:                                                   │
│  1. 定位原页面                                          │
│  2. 修改目标行                                          │
│  3. 标记页面脏                                          │
│                                                          │
│  策略 B: 删除+插入 (Delete+Insert)                      │
│  ─────────────────────────────────────────              │
│  适用条件:                                               │
│  - 更新的列包含索引键 (需要更新索引)                     │
│  - 行大小变化                                            │
│  流程:                                                   │
│  1. 删除原行 (标记删除)                                  │
│  2. 插入新行                                            │
│  3. 更新所有受影响索引                                   │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 4. DELETE 执行链路

### 4.1 DELETE 时序图

```
DELETE FROM users WHERE id = 1
    │
    ▼
┌─────────────────────────────────────────────┐
│              Executor                        │
│  DeleteExecutor::execute()                   │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 定位要删除的行                      │  │
│  │ 2. 记录删除信息到事务                 │  │
│  │ 3. 标记行为已删除                     │  │
│  │ 4. 从所有索引中移除                   │  │
│  │ 5. 写 WAL 日志                        │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Storage                         │
│  - 标记删除 (不立即回收空间)                  │
│  - 更新 B+Tree 索引                         │
│  - 写 WAL (Before Image + After Image)      │
└─────────────────────────────────────────────┘
```

## 5. SELECT 执行链路

### 5.1 SELECT 完整流水线

```
SELECT u.name, o.amount
FROM users u
JOIN orders o ON u.id = o.user_id
WHERE u.age > 18
ORDER BY o.amount DESC
LIMIT 100
    │
    ▼
┌─────────────────────────────────────────────┐
│              Parser                          │
│  └─► AST ──► Logical Plan                  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Planner                         │
│  Logical Plan:                             │
│  Limit(100)                                 │
│    └─► Sort(amount DESC)                   │
│          └─► HashJoin(u.id = o.user_id)    │
│                ├─► Filter(u.age > 18)       │
│                │     └─► TableScan(users)  │
│                └─► TableScan(orders)       │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Optimizer (CBO)                │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 统计信息收集 (cardinality)         │  │
│  │ 2. 代价计算 (CPU + IO)               │  │
│  │ 3. Join 次序优化                      │  │
│  │ 4. 物理算子选择                        │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Executor                        │
│  Volcano Iterator Model:                    │
│  Limit:100 ──► next()                     │
│     │                                      │
│  Sort:DESC ──► next()                     │
│     │                                      │
│  HashJoin ──► build() + probe()            │
│     │            │                         │
│  IndexScan  TableScan                      │
└─────────────────────────────────────────────┘
```
