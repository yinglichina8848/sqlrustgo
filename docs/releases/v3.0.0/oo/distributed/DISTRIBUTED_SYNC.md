## 3. 半同步复制 (Semi-sync Replication)

### 3.1 半同步原理

```
┌─────────────────────────────────────────────────────────────┐
│                    半同步复制原理                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  全异步 (Async):                                            │
│  ─────────────────────────────────────────────────────────  │
│  Master ────► Slave1 ────► Slave2 ────► Slave3           │
│       │                                                   │
│       └── 发送后立即返回，不等待确认                        │
│                                                              │
│  半同步 (Semi-sync):                                        │
│  ─────────────────────────────────────────────────────────  │
│  Master ────► Slave1 ────► Slave2 ────► Slave3           │
│       │           │                                         │
│       │           └──── 至少一个确认 ────► 返回成功        │
│       │                                                   │
│       └── 超时后退化为异步                                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 半同步配置

```rust
pub struct SemiSyncConfig {
    /// 是否启用半同步
    pub enabled: bool,
    /// 等待确认超时时间 (ms)
    pub ack_timeout_ms: u64,
    /// 等待确认的 slave 数量
    pub wait_count: usize,
    /// 超时后降级为异步
    pub degrade_on_timeout: bool,
}

impl Default for SemiSyncConfig {
    fn default() -> Self {
        Self {
            enabled: true,
            ack_timeout_ms: 10000,  // 10 秒
            wait_count: 1,
            degrade_on_timeout: true,
        }
    }
}
```

### 4.2 冲突检测与分配

```rust
pub struct MTScheduler {
    /// worker 数量
    num_workers: usize,
    /// 冲突检测器
    conflict_detector: ConflictDetector,
    /// 工作队列
    workers: Vec<Worker>,
}

impl MTScheduler {
    /// 为事件分配 worker
    fn assign_worker(&self, event: &BinlogEvent) -> usize {
        // 冲突类型
        let conflict_key = match event {
            BinlogEvent::Insert { table, key } => format!("{}.{}", table, key),
            BinlogEvent::Update { table, key, .. } => format!("{}.{}", table, key),
            BinlogEvent::Delete { table, key } => format!("{}.{}", table, key),
        };

        // 同一 key 的事件分配给同一 worker（保证顺序）
        let hash = self.hash(&conflict_key);
        hash % self.num_workers
    }
}
```

## 6. XA 事务 (两阶段提交)

### 6.1 XA 事务原理

```
┌─────────────────────────────────────────────────────────────┐
│                    XA 事务流程                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  应用                                                    │
│    │                                                       │
│    │  XA START 'tx1'                                     │
│    ▼                                                       │
│  ┌─────────────────────────────────────────┐              │
│  │           阶段 1: 执行                  │              │
│  │  ─────────────────────────────────────  │              │
│  │  执行 SQL1 (Shard1)                     │              │
│  │  执行 SQL2 (Shard2)                     │              │
│  │  执行 SQL3 (Shard3)                     │              │
│  └─────────────────────────────────────────┘              │
│    │                                                       │
│    │  XA END 'tx1'                                        │
│    ▼                                                       │
│  ┌─────────────────────────────────────────┐              │
│  │           阶段 2: 准备 (Prepare)        │              │
│  │  ─────────────────────────────────────  │              │
│  │  Shard1: XA PREPARE 'tx1' → OK         │              │
│  │  Shard2: XA PREPARE 'tx1' → OK         │              │
│  │  Shard3: XA PREPARE 'tx1' → OK         │              │
│  └─────────────────────────────────────────┘              │
│    │                                                       │
│    │  XA COMMIT 'tx1'                                     │
│    ▼                                                       │
│  ┌─────────────────────────────────────────┐              │
│  │           阶段 3: 提交 (Commit)         │              │
│  │  ─────────────────────────────────────  │              │
│  │  Shard1: XA COMMIT 'tx1' → OK         │              │
│  │  Shard2: XA COMMIT 'tx1' → OK         │              │
│  │  Shard3: XA COMMIT 'tx1' → OK         │              │
│  └─────────────────────────────────────────┘              │
│    │                                                       │
│    ▼                                                       │
│  事务完成                                                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 6.3 XA 恢复

```
┌─────────────────────────────────────────────────────────────┐
│                    XA 恢复流程                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  协调者崩溃恢复                                              │
│  ─────────────────────────────────────────────────────────  │
│  1. 读取持久化的协调者状态表                                │
│  2. 对于 PREPARED 状态的事务:                              │
│     - 如果所有参与者都提交 → 重新提交                        │
│     - 如果有任何参与者失败 → 回滚                            │
│  3. 对于 UNKNOWN 状态的事务:                               │
│     - 查询每个参与者的状态                                   │
│     - 根据多数决定提交或回滚                                 │
│                                                              │
│  参与者崩溃恢复                                              │
│  ─────────────────────────────────────────────────────────  │
│  1. 读取本地 WAL                                            │
│  2. 如果是 PREPARED → 等待协调者决定                       │
│  3. 如果是 COMMIT → 执行提交                               │
│  4. 如果是 ROLLBACK → 执行回滚                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```
