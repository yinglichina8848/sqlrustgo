## 2. WAL 协议原理

### 2.1 Write-Ahead Log 语义

```
┌─────────────────────────────────────────────────────────────┐
│                 Write-Ahead Log 协议                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  规则 1: 数据页修改前必须先写 WAL                            │
│  ─────────────────────────────────────────────────────────  │
│                                                              │
│  BEFORE:                                                    │
│  ┌─────────┐    ┌─────────┐                               │
│  │ Buffer  │ ──►│  Data   │                               │
│  │ Pool    │    │  Page   │                               │
│  └─────────┘    └─────────┘                               │
│                                                              │
│  AFTER (WAL):                                              │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐               │
│  │ Buffer  │ ──►│  WAL    │ ──►│  Data   │               │
│  │ Pool    │    │  Log    │    │  Page   │               │
│  └─────────┘    └─────────┘    └─────────┘               │
│                     │                                      │
│                     └── 必须先刷盘                           │
│                                                              │
│  规则 2: COMMIT 必须等 WAL 刷盘完成                          │
│  ─────────────────────────────────────────────────────────  │
│                                                              │
│  TX Commit:                                                 │
│  1. 写 BEGIN WAL Entry                                      │
│  2. 写数据修改                                              │
│  3. 写 COMMIT WAL Entry                                     │
│  4. 强制刷盘 (fsync)                                        │
│  5. 返回客户端 COMMIT 成功                                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 WAL 生命周期

```
事务 T1 开始                                              T1 提交
    │                                                         │
    │                                                         │
    ▼                                                         ▼
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│ BEGIN   │──►│ INSERT  │──►│ UPDATE  │──►│ UPDATE  │──►│ COMMIT  │
│ T1      │   │ row 1   │   │ row 2   │   │ row 3   │   │ T1      │
└─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘
    │                                                         │
    │                                                         │
    └─────────────────────────────────────────────────────────┘
                         WAL 文件
```

### 3.2 WAL Entry 结构

```rust
pub struct WalEntry {
    /// Log Sequence Number (全局递增)
    pub lsn: u64,
    /// 时间戳
    pub timestamp: u64,
    /// 事务 ID
    pub tx_id: u64,
    /// Entry 类型
    pub entry_type: WalEntryType,
    /// 表 ID
    pub table_id: u64,
    /// Row key (用于 update/delete)
    pub key: Option<Vec<u8>>,
    /// Row data (用于 insert/update)
    pub data: Option<Vec<u8>>,
}
```

### 3.3 Entry 类型

```rust
pub enum WalEntryType {
    Begin = 1,      // 事务开始
    Insert = 2,     // 插入行
    Update = 3,     // 更新行
    Delete = 4,     // 删除行
    Commit = 5,     // 事务提交
    Rollback = 6,   // 事务回滚
    Checkpoint = 7, // 检查点
    Prepare = 8,    // 2PC 准备阶段
}
```

### 6.2 恢复算法

```rust
fn recover(database: &Database) -> RecoveryResult {
    // 1. 读取检查点
    let checkpoint_lsn = read_checkpoint_lsn();

    // 2. 扫描 WAL，建立事务状态表
    let mut tx_states: HashMap<u64, TxState> = HashMap::new();

    for entry in scan_wal_from(checkpoint_lsn) {
        match entry.entry_type {
            WalEntryType::Begin => {
                tx_states.insert(entry.tx_id, TxState::Active);
            }
            WalEntryType::Commit => {
                if let Some(state) = tx_states.get_mut(&entry.tx_id) {
                    *state = TxState::Committed;
                }
            }
            WalEntryType::Rollback => {
                if let Some(state) = tx_states.get_mut(&entry.tx_id) {
                    *state = TxState::RolledBack;
                }
            }
            _ => { /* data entries */ }
        }
    }

    // 3. 重放已提交事务
    for entry in scan_wal_from(checkpoint_lsn) {
        if tx_states.get(&entry.tx_id) == Some(&TxState::Committed) {
            replay_entry(entry);
        }
    }

    // 4. 回滚未提交事务
    for (tx_id, state) in &tx_states {
        if *state == TxState::Active {
            rollback_transaction(*tx_id);
        }
    }
}
```

## 8. 性能优化

### 8.1 WAL 批量写入

```rust
// 批量写入优化
async fn write_entries(&self, entries: Vec<WalEntry>) -> SqlResult<()> {
    // 1. 收集到批次缓冲区
    let batch = entries.iter()
        .map(|e| e.to_bytes())
        .collect::<Vec<_>>();

    // 2. 批量追加到 WAL
    let mut writer = self.writer.lock().await;
    for bytes in batch {
        writer.write_all(&bytes)?;
    }

    // 3. 批量刷盘（组提交）
    if should_flush() {
        writer.flush()?;
    }

    Ok(())
}
```
