# TCL 执行链路

> Transaction Control Language: BEGIN, COMMIT, ROLLBACK, SAVEPOINT, SET TRANSACTION

## 1. TCL 概述

### 1.1 TCL 语句类型

```
┌─────────────────────────────────────────────────────────────┐
│                    TCL 语句分类                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────────┐  ┌─────────────────┐  ┌───────────┐ │
│  │ 事务开始/结束   │  │  事务控制      │  │ 隔离级别 │ │
│  │                 │  │                 │  │           │ │
│  │ BEGIN          │  │ COMMIT         │  │ SET TX   │ │
│  │ START TRANSACTION│  │ ROLLBACK      │  │           │ │
│  │                 │  │ SAVEPOINT      │  │           │ │
│  │                 │  │ RELEASE SP    │  │           │ │
│  │                 │  │ ROLLBACK TO   │  │           │ │
│  └─────────────────┘  └─────────────────┘  └───────────┘ │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.1 BEGIN 时序图

```
BEGIN

    ▼

│    command: BEGIN                           │
│  }                                          │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐

│  ┌─────────────────────────────────────┐  │

│  │ 2. 分配新的 TxId                    │  │
│  │ 3. 创建快照 (Snapshot)               │  │

    ▼
┌─────────────────────────────────────────────┐
│              WAL (可选)                        │
│  ┌─────────────────────────────────────┐  │
│  │ 写入 BEGIN WAL Entry                 │  │

## 4. COMMIT 执行链路

### 4.1 COMMIT 时序图

```
COMMIT
    │
    ▼
┌─────────────────────────────────────────────┐
│              Transaction Manager               │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 检查事务状态是否为 ACTIVE        │  │
│  │ 2. 生成提交时间戳                   │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              MVCC Engine                      │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 为所有修改的行更新 commit_ts     │  │
│  │ 2. 将事务状态设为 COMMITTED        │  │
│  │ 3. 清理未使用的版本                 │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              WAL Manager                      │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 写入 COMMIT WAL Entry            │  │
│  │ 2. 强制刷盘 (fsync)                │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Transaction Manager               │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 释放事务锁                      │  │
│  │ 2. 清空当前事务上下文              │  │
│  │ 3. 释放快照                        │  │

└─────────────────────────────────────────────┘

## 5. ROLLBACK 执行链路

### 5.1 ROLLBACK 时序图

```
ROLLBACK
    │
    ▼
┌─────────────────────────────────────────────┐
│              Transaction Manager               │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 检查事务状态                     │  │
│  │ 2. 获取 undo log                   │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              Undo Manager                     │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 逆序遍历 undo log               │  │
│  │ 2. 对每条记录:                     │  │
│  │    - INSERT → 删除行                │  │
│  │    - DELETE → 恢复行                │  │
│  │    - UPDATE → 恢复旧值             │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              MVCC Engine                      │
│  ┌─────────────────────────────────────┐  │
│  │ 1. 将事务状态设为 ROLLED_BACK      │  │
│  │ 2. 清理版本链                      │  │
│  │ 3. 标记版本为无效                  │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│              WAL Manager                      │
│  ┌─────────────────────────────────────┐  │
│  │ 写入 ROLLBACK WAL Entry              │  │
│  └─────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

## 7. MVCC 与事务

### 7.1 MVCC 可见性判断

```rust
impl TransactionContext {
    pub fn is_visible(&self, tx_id: TxId, commit_timestamp: Option<u64>) -> bool {
        match self.isolation_level {
            IsolationLevel::ReadCommitted => {
                // 读取已提交的数据
                self.snapshot.is_visible_read_committed(
                    tx_id,
                    commit_timestamp,
                    self.snapshot.snapshot_timestamp,
                )
            }
            IsolationLevel::RepeatableRead => {
                // 读取事务开始时的快照
                self.snapshot.is_visible(tx_id, commit_timestamp)
            }
            _ => self.snapshot.is_visible(tx_id, commit_timestamp),
        }
    }
}
```

### 8.2 死锁检测

```rust
pub struct DeadlockDetector {
    waits_for: HashMap<TxId, HashSet<TxId>>,
}

impl DeadlockDetector {
    /// 检测是否存在循环等待
    pub fn detect_deadlock(&self) -> Option<Vec<TxId>> {
        // DFS 检测环
        for tx_id in self.waits_for.keys() {
            let cycle = self.dfs(tx_id, &mut HashSet::new(), &mut Vec::new())?;
            if !cycle.is_empty() {
                return Some(cycle);
            }
        }
        None
    }
}
```

## 10. 覆盖率差距分析

### 10.1 当前覆盖率

| 组件 | 行覆盖率 | 说明 |

| manager.rs | ~85% | 事务管理器 |
