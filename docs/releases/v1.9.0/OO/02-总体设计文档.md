
### 3.2 模块职责

| 模块 | 职责 | 关键类/接口 |
|------|------|-------------|
| Parser | SQL 词法分析、语法解析 | Lexer, Parser, Statement |
| Planner | 逻辑计划生成、类型绑定 | LogicalPlanner, Binder |
| Optimizer | 查询重写、计划选择 | Optimizer, Rule, CostModel |
| Executor | 物理计划执行 | Executor, PhysicalPlan, Operator |
| Storage | 数据持久化、索引管理 | StorageEngine, BPlusTree, BufferPool |
| Transaction | 事务控制、并发管理 | TransactionManager, MVCC, LockManager |
| Server | 连接管理、会话管理 | ConnectionPool, Session |

## 4. 接口设计

### 4.1 Parser 接口

```rust
pub trait Parser {
    fn parse(&self, sql: &str) -> Result<Statement, SqlError>;
    fn parse_statements(&self, sql: &str) -> Result<Vec<Statement>, SqlError>;
}
```

### 4.2 Planner 接口

```rust
pub trait LogicalPlanner {
    fn create_plan(&self, statement: Statement) -> Result<LogicalPlan, SqlError>;
}

pub trait PhysicalPlanner {
    fn create_physical_plan(&self, logical_plan: &LogicalPlan) -> Result<Box<dyn PhysicalPlan>, SqlError>;
}
```

### 4.3 Optimizer 接口

```rust
pub trait Optimizer {
    fn optimize(&self, plan: LogicalPlan) -> Result<LogicalPlan, SqlError>;
}

pub trait OptimizerRule {
    fn apply(&self, plan: &mut LogicalPlan) -> Result<bool, SqlError>;
}

pub trait CostModel {
    fn estimate_cost(&self, plan: &PhysicalPlan) -> Cost;
}
```

### 4.4 Executor 接口

```rust
pub trait Executor {
    fn execute(&mut self, plan: &dyn PhysicalPlan) -> Result<ExecutorResult, SqlError>;
}

pub trait PhysicalPlan {
    fn name(&self) -> &str;
    fn execute(&self, ctx: &mut ExecutionContext) -> Result<ExecutorResult, SqlError>;
}
```

### 4.5 Storage 接口

```rust
pub trait StorageEngine {
    fn create_table(&mut self, info: &TableInfo) -> Result<(), SqlError>;
    fn drop_table(&mut self, name: &str) -> Result<(), SqlError>;
    fn insert(&mut self, table: &str, rows: Vec<Vec<Value>>) -> Result<usize, SqlError>;
    fn scan(&self, table: &str) -> Result<Vec<Vec<Value>>, SqlError>;
    fn delete(&mut self, table: &str, key: &[Value]) -> Result<usize, SqlError>;
    fn update(&mut self, table: &str, key: &[Value], values: &[Value]) -> Result<usize, SqlError>;
}
```

## 6. 事务管理设计

### 6.1 MVCC 实现

```uml
@startuml

class VersionStore {
  -versions: BTreeMap<RowKey, Vec<RowVersion>>
  --
  +get_visible_version(tx_id, key): Option<RowVersion>
  +add_version(key, value, tx_id)
  +gc_old_versions()
}

class RowVersion {
  -row_data: Vec<Value>
  -tx_id: TransactionId
  -begin_time: Timestamp
  -end_time: Option<Timestamp>
  -is_deleted: bool
}

class Transaction {
  -tx_id: TransactionId
  -isolation_level: IsolationLevel
  -read_set: Vec<RowKey>
  -write_set: Vec<RowKey>
  -state: TxState
  --
  +begin()
  +commit()
  +rollback()
  +is_visible(version): bool
}

Transaction --> VersionStore
VersionStore --> RowVersion

@enduml
```

### 6.2 隔离级别实现

| 隔离级别 | 实现方式 | 读一致性 |
|----------|----------|----------|
| READ UNCOMMITTED | MVCC | 最低，可能脏读 |
| READ COMMITTED | MVCC + 快照 | 每次读取获取新快照 |
| REPEATABLE READ | MVCC + 事务快照 | 事务内一致 |
| SERIALIZABLE | 锁 + MVCC | 最高 |

## 8. 查询优化设计

### 8.1 Cascades 优化器

```uml
@startuml

package "Cascades Optimizer" {
  
  [Memo] {
    -groups: Vec<Group>
    --
    +insert_expr()
    +merge_groups()
  }
  
  [Group] {
    -group_id: GroupId
    -logical_expressions: Vec<Expr>
    -best_cost: Cost
    -best_plan: PhysicalPlan
  }
  
  [Task Stack] {
    -tasks: Vec<OptimizeTask>
    --
    +push()
    +pop()
  }
  
  [Rule Engine] {
    -rules: Vec<OptimizeRule>
    --
    +match()
    +transform()
  }
  
  [Cost Model] {
    --
    +calculate_cost()
  }
  
  Memo --> Group
  Group --> Task
  Task --> Rule
  Rule --> Cost
  
}

@enduml
```

### 8.2 优化规则

| 规则名称 | 描述 | 适用场景 |
|----------|------|----------|
| PredicatePushdown | 谓词下推 | WHERE 条件提前 |
| ColumnPruning | 列裁剪 | SELECT 只读必要列 |
| JoinReordering | Join 重排 | 多表 Join 顺序 |
| IndexScanSelection | 索引选择 | 条件包含索引列 |
| AggregateSimplify | 聚合简化 | 常量聚合 |

## 10. 错误处理设计

### 10.1 错误分类

| 错误类型 | 错误码范围 | 处理策略 |
|----------|------------|----------|
| 语法错误 | 1000-1999 | 返回错误，终止执行 |
| 语义错误 | 2000-2999 | 返回错误，终止执行 |
| 运行时错误 | 3000-3999 | 事务回滚 |
| IO 错误 | 4000-4999 | 崩溃恢复 |
| 系统错误 | 5000-5999 | 进程退出 |

### 10.2 错误传播

```uml
@startuml

[SQL Client]
[Parser]
[Planner]
[Optimizer]
[Executor]
[Storage]

[SQL Client] --> [Parser]
[Parser] --> [Planner]
[Planner] --> [Optimizer]
[Optimizer] --> [Executor]
[Executor] --> [Storage]

note right of Storage
  错误向上传播
  每层可选择:
  - 转换错误
  - 包装错误
  - 重试操作
end note

@enduml
```
