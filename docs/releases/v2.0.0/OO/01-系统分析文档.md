## 2. 系统概述

### 2.1 系统背景

SQLRustGo 是一个用 Rust 实现的 SQL-92 兼容数据库管理系统，采用模块化分层架构。v2.0.0 版本是**分布式 RDBMS 里程碑版本**，实现完整的企业级数据库能力，包括分布式事务、列式存储、向量化执行等核心特性。

| 能力 | v1.9.0 | v2.0.0 |
|------|--------|--------|

```uml
@startuml

package "Domain Entities" {

  class Database {
    +name: String
    +tables: Vec<Table>

    +name: String

    +indexes: Vec<Index>

    +name: String

    +name: String

    +username: String
    +roles: Vec<Role>

    +name: String
```

## 8. 行为模型

### 8.1 查询执行流程

```uml
@startuml

|#LightBlue|Client|
|#LightGreen|Parser|
|#LightYellow|Planner|
|#LightCoral|Optimizer|
|#LightGray|Executor|
|#LightPink|Storage|

|Client| -> |Parser|: SQL Query
|Parser| -> |Parser|: Lexical Analysis
|Parser| -> |Parser|: Syntax Analysis
|Parser| -> |Planner|: AST
|Planner| -> |Planner|: Type Check
|Planner| -> |Planner|: Logical Plan
|Planner| -> |Optimizer|: Logical Plan
|Optimizer| -> |Optimizer|: Cost Optimization
|Optimizer| -> |Optimizer|: Rule Optimization
|Optimizer| -> |Executor|: Physical Plan
|Executor| -> |Executor|: Open
|Executor| -> |Storage|: Scan/Read
|Storage| --> |Executor|: RecordBatch
|Executor| -> |Executor|: Execute
|Executor| -> |Client|: Result
|Executor| -> |Executor|: Close

@enduml
```

### 8.2 2PC 分布式事务状态机

```
┌──────────────┐
│    INITIAL   │
└──────┬───────┘
       │ begin_transaction()
       ▼
┌──────────────┐
│  PREPARING   │
└──────┬───────┘
       │ all participants vote_prepare()
       ▼
┌──────────────┐     │ any participant votes_abort
│   PREPARED   │─────┘
└──────┬───────┘
       │ coordinator decides commit
       ▼
┌──────────────┐
│   COMMITTING  │
└──────┬───────┘
       │ all participants acknowledged
       ▼
┌──────────────┐
│   COMMITTED   │
└──────────────┘
```

## 9. 接口契约

### 9.1 StorageEngine Trait

```rust
pub trait StorageEngine: Send + Sync {
    fn scan(&self, table: &TableRef) -> Result<Box<dyn RecordBatchReader>>;
    fn insert(&self, table: &TableRef, batch: RecordBatch) -> Result<usize>;
    fn delete(&self, table: &TableRef, keys: &[Key]) -> Result<usize>;
    fn update(&self, table: &TableRef, keys: &[Key], batch: RecordBatch) -> Result<usize>;
    fn flush(&self) -> Result<()>;
    fn checkpoint(&self) -> Result<()>;
}
```

### 9.2 TransactionManager Trait

```rust
pub trait TransactionManager: Send + Sync {
    fn begin(&self, isolation: IsolationLevel) -> Result<TransactionId>;
    fn commit(&self, txid: TransactionId) -> Result<()>;
    fn rollback(&self, txid: TransactionId) -> Result<()>;
    fn get_status(&self, txid: TransactionId) -> Result<TransactionStatus>;
}
```

### 9.3 Coordinator Trait

```rust
pub trait Coordinator: Send + Sync {
    fn begin_distributed(&self, isolation: IsolationLevel) -> Result<TransactionId>;
    fn prepare(&self, txid: TransactionId) -> Result<PrepareResult>;
    fn commit(&self, txid: TransactionId) -> Result<CommitResult>;
    fn rollback(&self, txid: TransactionId) -> Result<()>;
}
```
