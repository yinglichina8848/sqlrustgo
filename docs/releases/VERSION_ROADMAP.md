# SQLRustGo 版本演化计划 v5.0

> **版本**: v5.0
> **更新日期**: 2026-03-21
> **制定依据**: SQL-92 功能覆盖路线图 + v2.0 技术范式明确
> **核心原则**: 性能和稳定性优先，SQL-92 全覆盖分阶段实现

---

## 一、版本号语义与成熟度目标

| 主版本 | 技术范式 | 成熟度等级 | 核心特征 | 对标系统 |
|--------|----------|------------|----------|----------|
| 1.x | 单机基础数据库 | L3 (Mini DBMS) | 火山模型、行存、B+Tree、MVCC、SQL-92 | early PostgreSQL |
| 2.x | 单机高性能分析引擎 | L4 (Analytical DB) | 向量化执行、列存、CBO、并行执行 | DuckDB |
| 3.x | 分布式数据库 | L5 (Distributed DB) | 分布式执行、分片、共识、分布式事务 | TiDB / CockroachDB |

**演进路径**: 1.x（可用）→ 2.x（快）→ 3.x（大）

## 四、v2.0 技术范式明确

### 4.1 v2.0 定位：单机高性能 + 全功能（平滑过渡）

```
v1.9 (全SQL-92)  →  v2.0 (高性能)
     │                    │
     │ 保留架构          │ 增强性能
     │ SQL全兼容         │ 向量化
     │                   │ 列存
     │                   │ CBO
     ↓                   ↓
┌─────────────────────────────────┐
│     v2.0: 单机高性能版          │
│  - 1.x 架构平移               │
│  - 性能大幅提升 (10-100x)     │
│  - 向量化 + 列存 + CBO         │
│  - 用户可立即使用全功能        │
└─────────────────────────────────┘
```

### 4.2 v2.0 vs v3.0 路径选择

| 选项 | 描述 | 优点 | 风险/缺点 |
|------|------|------|-----------|
| **选项A: 单机高性能** | 基于1.x架构，支持向量化、CBO、SQL-92完整 | 快速迭代，用户立即使用全功能 | 难以扩展分布式，吞吐受单机限制 |
| **选项B: 分布式** | 架构重构，支持多节点、分布式事务 | 可扩展到PB级，高吞吐 | 架构复杂，GA需要更多时间 |

**建议策略**:
- **v2.0 初期**: 单机高性能 + SQL-92 全功能（1.x → 2.0 平滑过渡）
- **v2.x 中期**: 可选向量化 + CBO + 并行查询增强
- **v2.x 后期 / v3.0**: 完整分布式架构（分布式事务、节点扩容）

## 六、成熟度演进路线

```
L1 (Toy)   →   L2 (Query Engine)   →   L3 (Mini DBMS)   →   L4 (Analytical DB)   →   L5 (Distributed DB)
                  v1.7                    v1.9                 v2.2                     v3.x
```

- v1.7 完成高并发和基础SQL-92，达到 L3
- v1.9 实现完整 SQL-92，达到 L3+
- v2.2 实现向量化、列存、CBO，达到 L4
- v3.x 具备分布式能力，达到 L5
