## 一、版本演进总览

```
v1.2.0 (2026-03) ──→ v1.3.0 (2026-04) ──→ v1.4.0 (2026-07) ──→ v1.5.0 (2026-10) ──→ v2.0.0 (2027-01)
   架构重构              架构稳定            SQL 能力完善          事务强化 + 向量化        向量化 + CBO
   (GA)                 (Draft)             (规划)              (规划)                 (规划)
```

**演进原则**: 先稳定再革命，逐步引入 2.0 组件，平滑过渡

## 七、风险与应对

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 向量化重写工作量大 | 高 | v1.5.0 原型验证，分模块替换 |
| CBO 复杂度高 | 高 | 先启发式+简单代价，逐步增强 |
| 统计信息影响性能 | 中 | 异步更新，按需收集 |
| 团队经验不足 | 中 | 培训，参考 DataFusion/DuckDB |
