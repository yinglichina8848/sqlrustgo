## 二、核心问题分析

### 问题 1: 手动逐个修复效率低下

**原因**:
- 没有批量处理思维
- 不熟悉 ast-grep/sed 批量替换

**改进方案**:
```bash
# 示例: 批量替换 releases 目录下的 CHANGELOG 路径
find docs/releases -name "*.md" -exec sed -i 's|../../CHANGELOG.md|../../../CHANGELOG.md|g' {} \;
```

### 问题 2: 未区分历史文档的特殊性

**观察**:
- `docs/releases/v1.1.0/` ~ `v1.2.0` 是 2026-03 的文档
- 当时的目录结构与现在不同
- 部分链接目标文件已删除或迁移

**结论**:
- 历史版本文档应标记 `[archived]`
- 而非全部修复
