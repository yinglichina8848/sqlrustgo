## 二、CBO 优化器架构

```
┌─────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────┤
│                                                          │
│  ┌──────────────┐    ┌──────────────┐                  │
│  │  Statistics  │───▶│  CostModel   │                  │
│  │   (stats.rs) │    │  (cost.rs)   │                  │
│  └──────────────┘    └──────────────┘                  │
│          │                   │                           │
│          ▼                   ▼                           │
│  ┌──────────────────────────────────────┐              │
│  │         Optimizer Rules                │              │
│  │  ┌──────────┐ ┌──────────┐ ┌─────┐ │              │
│  │  │IndexSelect│ │JoinOrder │ │Scan │ │              │
│  │  └──────────┘ └──────────┘ └─────┘ │              │
│  └──────────────────────────────────────┘              │
│                                                  │
└─────────────────────────────────────────────────────────┘
```

### 2.2 代价计算

| 操作 | 代价公式 |
|------|----------|
| SeqScan | rows × page_cost + cpu_tuple_cost |
| IndexScan | index_pages + index_selectivity × rows |
| HashJoin | build_cost + probe_cost |
| SortMergeJoin | sort_left + sort_right + merge |

## 三、Join 算法架构

### 3.1 算法选择

```
┌─────────────────────────────────────────────────────────┐
│                 Join Algorithm Selection                 │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Input ──▶ Check Join Type ──▶ Choose Algorithm        │
│                                      │                  │
│           ┌──────────────────────────┼──────────────┐   │
│           │                          │              │   │
│           ▼                          ▼              ▼   │
│     ┌──────────┐            ┌───────────┐    ┌────────┐ │
│     │ HashJoin │            │SortMerge  │    │Nested  │ │
│     │ Inner   │            │  Join     │    │ Loop   │ │
│     │ Left    │            │ Inner     │    │Cross   │ │
│     └──────────┘            │ Left      │    │Outer   │ │
│                            └───────────┘    └────────┘ │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 3.2 算法对比

| 算法 | 内存 | 适用场景 |
|------|------|----------|
| HashJoin | O(n) | 小表，无序 |
| SortMergeJoin | O(n log n) | 大表，有序 |
| NestedLoopJoin | O(1) | Cross Join |

## 四、向量化架构

### 4.1 SIMD 基础设施

```rust
// 向量化数据结构
pub struct Vector<T> {
    data: Vec<T>,
}

// 批量迭代器
pub trait BatchIterator {
    fn next_batch(&mut self, size: usize) -> Option<Batch>;
}
```

### 4.2 向量化流程

```
Raw Data ──▶ Parse ──▶ Vector ──▶ SIMD Ops ──▶ Result
```
