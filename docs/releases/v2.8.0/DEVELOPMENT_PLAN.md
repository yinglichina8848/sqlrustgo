# SQLRustGo v2.8.0 版本定义与开发计划

> 日期: 2026-05-02  
> 当前基线: `v2.7.0-GA` (commit `64ed7a5e`)  
> 开发分支: `develop/v2.8.0`  
> 目标版本: `v2.8.0` (Production+Distributed+Secure)  
> 目标定位: 达到 MySQL 5.7 功能覆盖率 92% + 初步分布式能力 + 安全评分 92%

---

## 1. v2.7.0 现状评估（用于 v2.8.0 立项）

基于 `docs/releases/v2.7.0/` 与当前仓库状态，v2.7.0 的主要结论：

1. MySQL 5.7 功能覆盖率已达 83%，核心 SQL DDL/DML 主路径基本可用。
2. WAL 崩溃恢复与事务隔离（RC/RR）已通过验证，支持备份恢复与审计证据链。
3. GMP 合规检索平台（全文检索 + 语义检索 + 图检索 + 混合检索）已形成服务面。
4. 生产门禁框架已形成，但缺乏分布式能力（横向扩展、跨节点一致性）。

结论：`v2.7.0` 已作为第一个 GA 生产版本发布。`v2.8.0` 应定位为"分布式增强 + 安全加固 + MySQL 兼容性进一步提升版本"。

## 2. 长期版本演进重定义（统一口径）

为避免现有路线图多口径冲突，统一如下：

1. `v2.6.0`: SQL 与内核收敛（功能正确性与门禁基线）
2. `v2.7.0`: 生产化版本（MySQL 5.7 级能力 + GMP 检索平台）
3. **`v2.8.0`: 分布式增强版本（横向扩展、跨节点一致性强化、安全加固）**
4. `v2.9.0`: 高级功能版本（并行查询、SQL 语料覆盖率提升、基准测试体系）
5. `v3.0.0`: 云原生/平台化版本（多租户、自动弹性、运维平台）

## 3. v2.8.0 版本定义

### 3.1 一句话定义

`v2.8.0` 是 SQLRustGo 的"分布式增强 + 安全加固"版本，在 v2.7.0 生产化基础上增加分区表、主从复制、故障转移、负载均衡、读写分离等核心分布式能力，并完成审计告警系统、列级权限、数据加密等安全特性。

### 3.2 目标能力边界

#### a) MySQL 5.7 功能对齐目标

| 指标 | v2.7.0 | v2.8.0 目标 | 变化 |
|------|--------|-------------|------|
| MySQL 5.7 功能覆盖率 | 83% | **92%** | +9% |
| 安全性评分 | 85% | **92%** | +7% |
| 多语言支持 | 40% | **65%** | +25% |
| 综合评分 | 83% | **90%** | +7% |

#### b) 兼容性增强

1. FULL OUTER JOIN — 基于 Hash 匹配算法实现完整外连接
2. TRUNCATE TABLE / REPLACE INTO — 语法解析与执行
3. 窗口函数完善 — ROW_NUMBER, RANK, DENSE_RANK
4. CHECK 约束 — 数据结构与解析支持（验证逻辑部分实现）
5. 分区表 — Range/List/Hash/Key 四类分区 + 分区裁剪优化

#### c) 分布式基础能力（核心增量）

1. 分区表（Range/List/Hash/Key）：75 测试通过
2. GTID 主从复制 + 半同步复制 + 复制延迟监控：79 测试通过
3. 故障转移（主节点宕机检测 < 5s + 自动切换 < 30s）：55 测试通过
4. 负载均衡（轮询策略 + 最少连接策略 + 健康检查）
5. 读写分离路由（SELECT → 从节点, INSERT/UPDATE → 主节点, 事务正确路由）：27 测试通过

#### d) 性能优化

1. SIMD 向量化加速 — `crates/vector/src/simd_explicit.rs`（5 测试通过）
2. Hash Join 并行化 — 代码存在但未集成到执行管线（推迟至 v2.9.0）
3. 查询计划器优化 — CBO 集成测试 12 测试 + planner 整体 81 测试通过

#### e) 安全加固

1. 审计告警系统 — SQL 执行审计、登录/登出审计、DDL/DML 追踪、安全事件告警（81 测试通过）
2. 列级权限控制 — ColumnMasker 数据结构存在，GRANT/REVOKE 解析器待实现（推迟至 v2.9.0）
3. 数据加密基础 — AES-256-GCM 加密模块代码存在但 feature-gated，未集成到存储管线（推迟至 v2.9.0）
4. SQL 防火墙 — SQL 注入检测、查询超时、行数限制、全表扫描检测
5. TLS 1.2/1.3 — 双向证书验证、自定义 CA

#### f) 文档与多语言

1. 英文错误消息（ERROR_MESSAGES.md）
2. 英文 API 文档（API_REFERENCE.md, API_USAGE_EXAMPLES.md）
3. 安全加固指南（SECURITY_HARDENING.md）

## 5. 备份恢复体系（v2.8.0）

### 5.1 存储引擎备份层

- **多格式导出**: CSV / JSON / SQL INSERT 语句
- **Point-in-Time Recovery (PITR)**: 支持三种恢复目标 — LSN(u64)、Timestamp(u64)、TransactionId(u64)
- **备份调度器**: 按日/周/月自动调度，可配置保留周期（默认 7 天），支持 gzip 压缩
- **备份存储后端**: 本地文件系统 / S3 兼容远程存储
- **WAL 集成**: 基于 Checkpoint 和 2PC Prepare 的一致恢复点标记

### 5.2 CLI 工具层

```bash
sqlrustgo-tools backup --database <db> --output-dir <dir> --backup-type full
sqlrustgo-tools restore --database <db> --backup-id <id> --backup-dir <dir>
```

### 5.3 备份验证

1. SHA-256 Checksum 完整性验证
2. Schema 元数据一致性验证
3. 行数计数匹配验证
4. WAL LSN 位置对齐验证
