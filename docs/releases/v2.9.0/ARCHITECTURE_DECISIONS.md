# 架构决策记录 (ADR)

> **版本**: v2.9.0
> **最后更新**: 2026-05-05

---

## 概述

本文档记录 SQLRustGo v2.9.0 开发过程中的关键架构决策。

---

## ADR-001: 使用 Rust 作为核心开发语言

**状态**: 已接受

**背景**: 选择 Rust 以获得内存安全、高性能和并发支持。

**决策**: 使用 Rust 2021 edition，Tokio 异步运行时。

---

## ADR-002: 使用 B+Tree 作为默认索引结构

**状态**: 已接受

**背景**: 需要高效的点查询和范围查询支持。

**决策**: 使用 B+Tree 作为主索引结构，支持聚簇索引和二级索引。

---

## ADR-003: WAL 预写日志

**状态**: 已接受

**背景**: 确保崩溃恢复能力。

**决策**: 实现 WAL 机制，检查点间隔可配置。

---

## ADR-004: MVCC 事务隔离

**状态**: 已接受

**背景**: 支持并发读写。

**决策**: 使用 MVCC + SSI（Serializable Snapshot Isolation）隔离级别。

---

## ADR-005: 分布式复制架构

**状态**: 已接受

**背景**: v2.9.0 聚焦分布式架构。

**决策**: 实现 semi-sync 复制、MTS 并行复制、多源复制。

---

## ADR-006: CTE 和窗口函数支持

**状态**: 已接受

**背景**: 提升 SQL 兼容性。

**决策**: 使用递归 CTE 实现，窗口函数使用 Apache Calcite 设计。

---

## ADR-007: Coverage Pipeline 双工具策略

**状态**: 已接受

**背景**: 需要行覆盖率和分支覆盖率。

**决策**: 使用 cargo-tarpaulin（行覆盖率）+ cargo-llvm-cov（分支+路径覆盖率）。

---

## ADR-008: Gate OS Lite B-Gate 设计

**状态**: 已接受

**背景**: 需要轻量级 CI gate 系统。

**决策**: Nomad runner + Gitea Actions，B-Gate 验证，L2 门禁可选。

---

## ADR-009: Proof Registry 形式化验证

**状态**: 已接受

**背景**: 建立可验证的正确性证明。

**决策**: TLA+ / Dafny / Formulog 三工具验证，Proof Registry 统一管理。
