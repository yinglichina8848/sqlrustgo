## 二、AI-CLI 协作邀请

### 2.1 协作目标

修复 DeepSeek 评估发现的问题，使 v1.1.0-draft 达到 L3 产品级标准。

### 2.2 任务分配

|人工智能代理| 任务 | 优先级 |
|----------|------|--------|
|**心敞开**|Clippy 错误修复、executor unwrap 替换| P0 |
|**敞开心扉**| WHERE AND/OR、表达式求值、多列索引 | P0 |
|**维护者**| 代码审核、测试验证 | P1 |

### 2.3 协作规范

AI-CLI 工具参与协作时，请遵循：

1. **身份规范**: 参考 [AI Agent 提示词体系](../AI增强软件工程/AI_AGENT_PROMPTS.md)
2. **分支规范**: 基于 `release/v1.1.0` 创建 `fix/<issue>` 分支
3. **PR 规范**: 目标分支为 `release/v1.1.0`，非 `main`
4. **审核规范**: 作者不能审核自己的 PR

## 六、协作流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          AI-CLI 协作流程                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│   1. AI-CLI 接收任务                                                         │
│      └── 参考 AI_AGENT_PROMPTS.md 选择身份                                  │
│                                                                              │
│   2. 创建修复分支                                                            │
│      └── git checkout -b fix/<issue> origin/release/v1.1.0                  │
│                                                                              │
│   3. 实现修复                                                                │
│      └── 遵循代码规范，替换 unwrap，添加测试                                 │
│                                                                              │
│   4. 创建 PR                                                                 │
│      └── 目标分支: release/v1.1.0                                           │
│                                                                              │
│   5. 等待审核                                                                │
│      └── maintainer 审核通过后合并                                           │
│                                                                              │
│   6. 验证                                                                    │
│      └── 运行 CI，确认门禁通过                                               │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```
