## 架构重构说明

### 目录重构进展 (新增)

- 已完成 `src/` 向 `crates/` workspace 的核心迁移。
- 已建立 executor / optimizer / planner / storage 等 crate 的模块化边界。
- 目录重构相关改进作为 v1.2.0 Alpha 阶段基础设施保留，不删除历史记录。

### v1.2.0 架构变化

```
v1.1.x 架构                    v1.2.0 架构 (抽象化)
┌─────────────┐               ┌─────────────────────┐
│ FileStorage │               │ StorageEngine trait │
└─────────────┘               │   ├── FileStorage   │
                              │   └── MemoryStorage │
┌─────────────┐               └─────────────────────┘
│ RowExecutor │               ┌─────────────────────┐
└─────────────┘               │  VectorizedExecutor │
                              │  (基于 RecordBatch) │
┌─────────────┐               └─────────────────────┘
│ SimpleStats │               ┌─────────────────────┐
└─────────────┘               │  CostModel trait    │
                              │  + StatsCollector   │
┌─────────────┐               └─────────────────────┘
│ Hardcoded   │               ┌─────────────────────┐
│ Optimizer   │               │ Optimizer (Memo)    │
└─────────────┘               └─────────────────────┘
```

### 接口抽象层

| 模块 | v1.1.0 | v1.2.0 |
|------|--------|--------|
| 存储 |FileStorage (具体实现)|存储引擎特征|
| 执行 |RowExecutor (行式)|VectorizedExecutor（列式）+ RecordBatch|
| 统计 | 简单计数器 |表统计 + 列统计 + 统计收集器|
| 优化 | 硬编码规则 |CostModel trait + Memo 结构|
| 网络 | 同步处理 | 异步框架预留 |

### 为 v2.0 做准备

v1.2.0 的接口抽象为 v2.0 分布式架构打基础：

- **StorageEngine**: 可扩展为分布式存储
- **RecordBatch**: 适合网络传输
- **CostModel**: 可扩展网络成本计算
- **Memo**: 支持分布式优化
