## 一、概述

### 1.1 目标

实现 Clustered Index (聚集索引) 优化，提升范围查询和主键查找性能：

- **数据物理排序**: 主键顺序决定数据物理存储顺序
- **范围查询优化**: 减少 I/O 操作
- **主键查找加速**: B+树叶子节点直接存储完整行数据

### 1.2 核心理念

```
Clustered Index = 主键排序 + 数据行存储 + B+树结构
```

## 三、实现细节

### 3.1 存储结构

```rust
/// 聚集索引叶子节点
pub struct ClusteredLeafNode {
    /// 主键值
    pub key: Vec<u8>,
    /// 行数据 (所有列值连续存储)
    pub row_data: Vec<u8>,
    /// 下一个叶子节点指针
    pub next_leaf: Option<PageId>,
}
```

### 3.2 创建聚集索引

```sql
-- 主键自动创建聚集索引
CREATE TABLE orders (
    id INTEGER PRIMARY KEY,  -- 聚集索引
    customer_id INTEGER,
    amount INTEGER
);

-- 显式创建聚集索引
CREATE CLUSTERED INDEX idx_orders_id ON orders(id);
```

### 3.3 查询优化

| 查询类型 | 优化效果 |
|----------|----------|
| `SELECT * FROM orders WHERE id = 123` | 单次 I/O 获取完整行 |
| `SELECT * FROM orders WHERE id BETWEEN 100 AND 200` | 顺序扫描叶子节点 |
| `SELECT id FROM orders WHERE id > 500` | 仅索引扫描 |

## 四、约束与限制

| 限制 | 说明 |
|------|------|
| 每个表一个聚集索引 | InnoDB 风格 |
| 主键优先 | 未指定时使用第一个唯一索引 |
| 列限制 | 不支持 BLOB/TEXT 列作为主键 |

## 五、性能分析

### 5.1 基准测试

| 操作 | 聚集索引 | 非聚集索引 |
|------|----------|------------|
| 主键查找 | 1 I/O | 2 I/O (索引 + 数据) |
| 范围扫描 | 顺序 I/O | 随机 I/O |
| 排序 | 无额外成本 | 需要回表 |
