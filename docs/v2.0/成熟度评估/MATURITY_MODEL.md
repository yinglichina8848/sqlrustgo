## 一、成熟度等级定义

我们定义 5 个等级：

| 等级 | 名称 | 说明 |
|:-----|:-----|:-----|
| L0 | 实验级 | Prototype |
| L1 | 可运行级 | Working Project |
| L2 | 工程化级 | Engineering Level |
| L3 | 产品级 | Production Ready |
| L4 | 企业级 | Enterprise Grade |

## 四、L2 → L3 升级路径

### 4.1 必须完成

| 任务 | 说明 | 优先级 |
|:-----|:-----|:-------|
| 架构稳定性评审 | 模块边界清晰 | P0 |
| API 冻结策略 | 稳定 API 承诺 | P0 |
| 性能基准测试 | 建立性能基线 | P0 |
| 文档站点 | 用户文档 | P1 |
| 插件机制设计 | 可扩展架构 | P1 |
| 代码所有权制度 | CODEOWNERS | P2 |

### 4.2 检查清单

```
L2 → L3 升级检查清单：

架构：
[ ] 模块依赖图清晰
[ ] 无循环依赖
[ ] 每个模块可独立测试

API：
[ ] API 文档完整
[ ] 版本兼容策略
[ ] 升级迁移指南

质量：
[ ] 覆盖率 ≥ 80%
[ ] 性能基准测试
[ ] 无 blocker bug

文档：
[ ] 用户手册
[ ] API 参考
[ ] 架构设计文档

社区：
[ ] 贡献指南
[ ] 行为准则
[ ] Issue 模板
```
