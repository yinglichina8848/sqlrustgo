### 阶段 2: 文档表结构 (P0)

**目标**: 实现文档相关表

#### 2.1 文档主表 (documents)

```sql
CREATE TABLE documents (
    id UUID PRIMARY KEY,
    title TEXT NOT NULL,
    doc_type ENUM('SOP', '规范', '审核', '报告', '日志') NOT NULL,
    version TEXT,
    status ENUM('draft', 'approved', 'archived') DEFAULT 'draft',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    effective_date DATE,
    department TEXT,
    author TEXT
);
```

#### 2.2 文档内容表 (document_contents)

```sql
CREATE TABLE document_contents (
    id UUID PRIMARY KEY,
    doc_id UUID REFERENCES documents(id) ON DELETE CASCADE,
    section TEXT,
    content TEXT,
    content_hash TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE FULLTEXT INDEX idx_doc_content ON document_contents(content);
```

#### 2.3 关键字表 (document_keywords)

```sql
CREATE TABLE document_keywords (
    id UUID PRIMARY KEY,
    doc_id UUID REFERENCES documents(id) ON DELETE CASCADE,
    keyword TEXT,
    source ENUM('manual', 'auto_extract') DEFAULT 'manual',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_keywords ON document_keywords(keyword);
```

### 阶段 4: 向量计算函数 (P0)

**目标**: 实现向量相似度计算

#### 4.1 内置函数

```rust
// crates/functions/src/vector.rs

/// 余弦相似度
pub fn cosine_similarity(a: &[f32], b: &[f32]) -> f32 {
    let dot = a.iter().zip(b.iter()).map(|(x, y)| x * y).sum::<f32>();
    let norm_a = a.iter().map(|x| x * x).sum::<f32>().sqrt();
    let norm_b = b.iter().map(|x| x * x).sum::<f32>().sqrt();
    dot / (norm_a * norm_b)
}

/// 内积
pub fn inner_product(a: &[f32], b: &[f32]) -> f32 {
    a.iter().zip(b.iter()).map(|(x, y)| x * y).sum()
}

/// 欧氏距离
pub fn euclidean_distance(a: &[f32], b: &[f32]) -> f32 {
    a.iter().zip(b.iter()).map(|(x, y)| (x - y).powi(2)).sum::<f32>().sqrt()
}
```

#### 4.2 SQL 函数接口

```sql
-- 余弦相似度
SELECT cosine_similarity(embedding, '[0.1, 0.2, ...]') FROM document_embeddings;

-- 向量检索 (简化的 top-k)
SELECT * FROM document_embeddings
ORDER BY cosine_similarity(embedding, query_vector) DESC
LIMIT 10;
```

### 阶段 5: OpenClaw SQL API (P1)

**目标**: 实现 HTTP API 接口

#### 5.1 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/sql_exec` | POST | 执行 SQL |
| `/api/v1/sql_batch` | POST | 批量操作 |
| `/api/v1/vector_embed` | POST | 生成嵌入向量 |
| `/api/v1/vector_search` | POST | 向量检索 |

#### 5.2 API 格式

```json
// sql_exec
POST /api/v1/sql_exec
{
  "sql": "SELECT * FROM documents WHERE doc_type = 'SOP'",
  "params": {},
  "options": {
    "parallel_degree": 4,
    "timeout_ms": 5000
  }
}

// vector_search
POST /api/v1/vector_search
{
  "query": "GMP 清洁验证标准操作流程",
  "table": "document_embeddings",
  "top_k": 10,
  "filters": {
    "doc_type": ["SOP", "规范"],
    "status": ["approved"]
  },
  "score_threshold": 0.7
}
```

1. **类型系统** (最关键，被其他所有模块依赖)

5. **全文检索** (基于现有 FullTextIndex)
6. **OpenClaw API** (独立网络层)
