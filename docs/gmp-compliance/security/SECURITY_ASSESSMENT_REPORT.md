### 4.2 详细分析

#### SEC-001: 无 MFA 认证

**描述**: 系统不支持多因素认证，依赖单一密码认证。

**风险**: 如果密码泄露，攻击者可完全访问数据库。

**缓解措施**:
- 强制 TLS 传输
- 强密码策略 (12+ 字符)
- 账户锁定 (5 次失败后)
- 审计所有登录尝试

**CVSS**: 6.5 (Medium)

```
CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:N
```

**建议**: v3.2.0 实现 TOTP MFA

---

#### SEC-005: 审计存储未加密

**描述**: 审计日志以明文存储在磁盘上。

**风险**: 如果磁盘被物理窃取，审计内容暴露。

**缓解措施**:
- OS 级文件加密 (dm-crypt)
- 文件系统权限 700
- 物理安全
- Append-only 权限

**CVSS**: 7.1 (High)

```
CVSS:3.1/AV:P/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:N
```

**建议**: v3.2.0 实现 AES-256-GCM 存储加密
