## 二、知识回顾：动态建模与架构图

### 2.1 顺序图

- **对象**：参与交互的实例
- **生命线**：对象的时间线
- **消息**：同步、异步、返回
- **激活条**：对象执行操作的时间段
- **组合片段**：alt（分支）、loop（循环）、opt（可选）、par（并行）

### 2.2 状态图

- **状态**：对象生命周期中的阶段
- **事件**：触发状态转换的事情
- **转换**：状态之间的迁移
- **内部活动**：entry/do/exit

### 2.3 活动图

- **活动节点**：执行的动作
- **控制节点**：决策、分叉、汇合
- **泳道**：划分责任区域

### 2.4 组件图与部署图

- **组件图**：系统的物理模块及接口依赖
- **部署图**：节点上的组件部署及网络连接

## 三、操作步骤

### 步骤1：使用AI生成顺序图（20分钟）

#### 1.1 场景描述

以图书管理系统的"借书"流程为例：

用户（学生）发起借书请求，系统验证用户身份和图书可借性，创建借阅记录，更新图书状态，返回借书成功信息。

#### 1.2 生成顺序图

在Trae IDE的AI对话中输入：

```
请为"图书管理系统"的借书流程生成顺序图，使用PlantUML语法。
参与者：学生（Actor）、系统界面（UI）、借书控制器（BorrowController）、图书服务（BookService）、借阅服务（BorrowService）、图书数据库（BookDB）、借阅数据库（BorrowDB）

流程：
1. 学生点击借书按钮，输入图书ID
2. UI调用控制器的borrowBook(bookId, userId)方法
3. 控制器调用BookService的getBook(bookId)验证图书是否存在且可借
4. BookService查询BookDB返回图书信息
5. 控制器调用BorrowService的createBorrowRecord(userId, bookId)
6. BorrowService在BorrowDB中插入记录
7. 控制器调用BookService的updateBookStatus(bookId, 'borrowed')
8. BookService更新BookDB中的状态
9. 控制器返回借书成功信息给UI
10. UI显示借书成功

使用alt片段处理图书不可借或用户违规的情况。
```

#### 1.3 检查并优化

- 确认消息顺序正确
- 确认激活条使用合理
- 可要求AI添加返回消息（可选）

### 步骤2：使用AI生成状态图（20分钟）

#### 2.1 场景描述

以"图书"对象的状态变化为例：

图书有"可借"、"已借出"、"遗失"、"报废"等状态，状态转换由事件触发。

#### 2.2 生成状态图

在AI对话中输入：

```
请为"图书"对象生成状态图，使用PlantUML语法。
状态：可借（Available）、已借出（Borrowed）、遗失（Lost）、报废（Discarded）
初始状态：可借
终止状态：报废
转换事件：
- 借书：Available → Borrowed
- 还书：Borrowed → Available
- 报失：Borrowed → Lost
- 找到：Lost → Available
- 报废：Available/Lost → Discarded
在Available状态添加entry动作：打印"图书可借"
在Borrowed状态添加do动作：计时借阅天数
```

**✅ 检查点2：截图保存状态图**

### 步骤3：使用AI生成活动图（20分钟）

#### 3.1 场景描述

以图书管理系统的"还书"流程为例，涉及多个角色并行操作。

#### 3.2 生成活动图

在AI对话中输入：

```
请为"图书管理系统"的还书流程生成活动图，使用PlantUML语法。
泳道：学生、系统、仓库（可选）
流程：
- 学生提交还书请求
- 系统验证借阅记录
- 如果超期，计算罚款

   - 仓库（如果有）接收图书
- 系统通知学生还书成功
使用决策节点处理超期情况。
```

### 步骤4：使用AI生成组件图（15分钟）

#### 4.1 系统模块划分

图书管理系统的模块：

- 用户界面模块（Web/App）
- 业务逻辑模块（借书、还书、查询）
- 数据访问模块
- 数据库

#### 4.2 生成组件图

在AI对话中输入：

```
请为"图书管理系统"生成组件图，使用PlantUML语法。
组件：
- 前端应用（Frontend）
- 借书服务（BorrowService）
- 还书服务（ReturnService）
- 查询服务（QueryService）
- 用户服务（UserService）
- 数据访问层（DataAccess）
- 数据库（Database）

关系：
- 前端依赖所有服务
- 服务依赖数据访问层
- 数据访问层依赖数据库
- 借书服务和还书服务可能依赖用户服务
标注接口（提供的和需要的）。
```

## 六、AI工具使用技巧总结

### 6.1 生成动态图的Prompt要点

- 明确参与者/对象（顺序图）
- 明确状态和事件（状态图）
- 明确泳道和流程步骤（活动图）
- 明确组件和依赖（组件图）
- 明确节点和连接（部署图）
- 指定输出格式（PlantUML优先）

### 6.2 常见问题的修正

- 顺序图缺少激活条：要求AI添加 activate / deactivate
- 状态图缺少初始/终止状态：补充说明
- 活动图缺少并发：用 fork / join 描述
- 组件图接口不明确：要求标注 provided interface / required interface

### 6.3 组合使用

可以先让AI生成一个初步图，然后逐步添加细节，例如：

```
在上一个顺序图基础上，添加一个超时处理片段：如果数据库响应超过2秒，返回超时错误。
```
