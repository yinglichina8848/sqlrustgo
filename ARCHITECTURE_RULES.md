# SQLRustGo AI 协作安全规则

> **版本**: 1.0
> **制定日期**: 2026-03-06
> **制定人**: yinglichina8848
> **适用对象**: 所有 AI 协作开发

---

## 一、核心原则

**AI 协作开发必须遵循版本阶段规则，禁止绕过任何门禁**

---

## 二、阶段限制规则

### 2.1 Draft 阶段

| 操作 | 权限 | 说明 |
|------|------|------|
| 目录结构调整 | ✅ 允许 | 可以重构 crate 结构 |
| 模块拆分 | ✅ 允许 | 可以拆分/合并模块 |
| 接口设计 | ✅ 允许 | 可以设计新接口 |
| API 变更 | ✅ 允许 | 可以修改 API 签名 |
| 依赖升级 | ✅ 允许 | 可以升级依赖版本 |

**AI 行动指南**:
- 可以自由进行架构设计
- 必须确保编译通过
- 建议创建架构设计文档

### 2.2 Alpha 阶段

| 操作 | 权限 | 说明 |
|------|------|------|
| 新功能开发 | ✅ 允许 | 可以添加新功能 |
| 新模块添加 | ✅ 允许 | 可以添加新模块 |
| 目录结构调整 | ❌ 禁止 | 架构已冻结 |
| 接口签名变更 | ⚠️ 限制 | 需要评审 |
| 依赖升级 | ⚠️ 限制 | 需要评估影响 |

**AI 行动指南**:
- 专注于功能实现
- 不要修改目录结构
- 接口变更需要说明理由

### 2.3 Beta 阶段

| 操作 | 权限 | 说明 |
|------|------|------|
| Bug 修复 | ✅ 允许 | 可以修复 Bug |
| 性能优化 | ✅ 允许 | 可以优化性能 |
| 测试补充 | ✅ 允许 | 可以添加测试 |
| 新功能 | ❌ 禁止 | 功能已冻结 |
| 新模块 | ❌ 禁止 | 功能已冻结 |
| 接口变更 | ❌ 禁止 | 接口已冻结 |
| 依赖升级 | ❌ 禁止 | 需要稳定环境 |

**AI 行动指南**:
- 只修复 Bug 和优化性能
- 不要添加新功能
- 不要修改接口

### 2.4 RC 阶段

| 操作 | 权限 | 说明 |
|------|------|------|
|Critical Bug 修复| ✅ 允许 | 仅限关键 Bug |
| 其他 Bug 修复 | ❌ 禁止 | 推迟到下一版本 |
| 性能优化 | ❌ 禁止 | 代码已冻结 |
| 重构 | ❌ 禁止 | 代码已冻结 |
| 测试修改 | ⚠️ 限制 | 仅修复测试 Bug |

**AI 行动指南**:
- 只修复 Critical Bug
- 任何修改需要说明紧急性
- 不做任何非必要修改

### 2.5 GA 阶段

| 操作 | 权限 | 说明 |
|------|------|------|
| 任何代码修改 | ❌ 禁止 | 版本已发布 |
| 文档更新 | ✅ 允许 | 可以更新文档 |
| 创建 Tag | ❌ 禁止 | 需要人工操作 |

**AI 行动指南**:
- 不要修改任何代码
- 可以更新文档
- 等待下一版本开发

## 三、目录结构保护规则

### 3.1 受保护目录

以下目录结构在 **Alpha 阶段后禁止修改**:

```
src/
├── lexer/          # 词法分析器 - Alpha 后禁止移动
├── parser/         # 语法分析器 - Alpha 后禁止移动
├── planner/        # 查询规划器 - Alpha 后禁止移动
├── optimizer/      # 优化器 - Alpha 后禁止移动
├── executor/       # 执行器 - Alpha 后禁止移动
├── storage/        # 存储引擎 - Alpha 后禁止移动
├── transaction/    # 事务管理 - Alpha 后禁止移动
├── catalog/        # 目录管理 - Alpha 后禁止移动
├── auth/           # 认证授权 - Alpha 后禁止移动
├── network/        # 网络层 - Alpha 后禁止移动
├── monitoring/     # 监控 - Alpha 后禁止移动
├── types/          # 类型系统 - Alpha 后禁止移动
└── error/          # 错误处理 - Alpha 后禁止移动
```

### 3.2 目录操作规则

| 阶段 | 新增目录 | 移动目录 | 删除目录 | 重命名目录 |
|------|----------|----------|----------|------------|
| Draft | ✅ | ✅ | ✅ | ✅ |
| Alpha | ✅ | ❌ | ❌ | ❌ |
| Beta | ❌ | ❌ | ❌ | ❌ |
| RC | ❌ | ❌ | ❌ | ❌ |
| GA | ❌ | ❌ | ❌ | ❌ |

## 五、AI 操作限制清单

### 5.1 绝对禁止操作

以下操作 **任何阶段都禁止 AI 直接执行**:

| 操作 | 原因 |
|------|------|
|直接 Push 到 main| 绕过 PR 流程 |
|强力推| 破坏历史记录 |
| 删除分支 | 可能丢失代码 |
|修改 Branch Protection| 安全风险 |
| 创建/删除 Tag | 需要人工确认 |
| 合并 PR (无审核) | 绕过审核流程 |
| 修改 CI 配置 | 安全风险 |
| 添加/删除 Secrets | 安全风险 |

### 5.2 需要确认的操作

以下操作需要 **人工确认后才能执行**:

| 操作 | 确认方式 |
|------|----------|
| 合并 PR | 等待审核通过 |
|创建 Release| 人工批准 |
| 修改依赖版本 | 评估影响 |
| 修改 API 签名 | 架构评审 |
| 添加新模块 | 功能评审 |

## 六、AI 行动检查表

### 6.1 每次修改前检查

```
□ 当前是什么阶段？
□ 这个修改在这个阶段允许吗？
□ 是否需要人工确认？
□ 是否会影响受保护的目录/接口？
□ 是否通过了 CI？
```

### 6.2 提交前检查

```
□ 编译是否通过？
□ 测试是否通过？
□ Clippy 是否通过？
□ 是否有文档更新？
□ Commit message 是否清晰？
```

## 七、违规处理

### 7.1 违规类型

| 级别 | 违规类型 | 处理方式 |
|------|----------|----------|
| 🔴 严重 | 绕过门禁、直接 Push | 立即回滚、禁用权限 |
| 🟠 中等 | 阶段违规修改 | 回滚修改、重新评审 |
| 🟡 轻微 | 未更新文档 | 补充文档 |

### 7.2 恢复流程

```
发现问题
    │
    ├── 评估影响范围
    │
    ├── 回滚违规修改
    │
    ├── 分析违规原因
    │
    ├── 更新规则文档
    │
    └── 重新执行正确流程
```
