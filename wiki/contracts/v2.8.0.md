## Rule Definitions (Summary)

| Rule | Statement | Enforcement |
|------|-----------|------------|
| R1 | No test file modification | `git diff --name-only \| grep "^tests/"` |
| R2 | No new `#[ignore]` | `git diff \| grep "^+.*#\[ignore\]"` |
| R3 | Proof must be CI-generated | CI regenerates, PR cannot submit |
| R4 | All tests must pass | `cargo test` exit code = 0 |
| R5 | Baseline must be verified | `baseline_verified = true` |
| R6 | Test count cannot decrease | `passed >= 226` |
| R7 | CI cannot be modified | `ci_self_test.sh` (out-of-band) |

## Attack Vectors

```
AV1: Test Immutability Bypass via Macro — HIGH — Not Mitigated
AV2: Partial Test Execution — HIGH — Not Mitigated
AV3: CI Workflow Disablement — CRITICAL — Mitigated (T5)
AV4: Verification Engine Tampering — CRITICAL — Partially Mitigated (R3)
AV5: Semantic Regression — CRITICAL — Not Mitigated (by design)
AV6: Ignore Semantic Bypass — MEDIUM — Partially Mitigated (R2)
AV7: Baseline Poisoning — HIGH — Not Mitigated
AV8: Proof Replay Attack — LOW — Fully Mitigated (R3)
AV9: Non-Deterministic Test Masking — MEDIUM — Not Mitigated
```

---

## Limitations

```
L1: Semantic Correctness Gap
    passes(tests) ⊭ correct(code)
    Tests verify behavior, not relational semantics.

L2: Trust Assumption Chain
    Proof validity → T1 ∧ T2 ∧ T3 ∧ T4 ∧ T5 ∧ T6
    If any Ti is false → Proof invalid

L3: CI Runner Compromise
    No technical defense within system.
    Requires organizational security controls.
```

```
GIVEN:
  G1: Rules R1-R7 enforced by CI
  G2: CI execution environment trusted (T1-T6)
  G3: Test suite T executes completely (226 tests)
  G4: Proof P generated by V in CI

CONCLUDED:
  C1: PR does not modify tests/ (R1)
  C2: PR does not introduce new #[ignore] (R2)
  C3: Proof P is CI-generated, not PR-submitted (R3)
  C4: All 226 tests pass (R4)
  C5: baseline_verified = true (R5)
  C6: Test coverage maintained (R6)

NOT CONCLUDED:
  U1: Code is semantically correct (L1)
  U2: CI runner is not compromised (L2)
  U3: verification_engine.py has no bugs (L4)
```
